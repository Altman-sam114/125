# v5.8j 检查面板 raw id 与目标状态硬化记录

本文记录唐宋 v5.8j 小切片：在 v5.8i 命令反馈与战报元数据硬化后，继续收口军队/州府检查面板中玩家可见的 raw id 与英文目标状态。

## 目标

- 延续 v5.8 发布候选硬化，减少唐宋默认主路径中 `UnitInspectorView` / `RegionInspectorView` 的 `RegionId`、`TheaterId`、`FrontZoneId`、`FrontLineId`、补给源 id 和英文 `None/controlled` 暴露。
- 保持底层 raw id、Codable schema、规则、命令、补给和地图权威不变，只做 UI 派生显示桥。
- 继续遵守用户要求：不跑本地测试、build、Swift parse、Markdown 检查或 `git diff --check`，推送后只由 GitHub Actions 云端验证。

## 非目标

- 不改 `RegionId` / `TheaterId` / `FrontZoneId` / `FrontLineId` 类型和存档格式。
- 不改 `SupplyRules`、`VictoryRules`、`RuleEngine`、`CommandValidator`、`CommandExecutor` 或任何控制权/补给/目标判定。
- 不做完整 accessibility、截图、布局、VoiceOver 或发布候选全量审计。
- 不处理 legacy 阿登路径的英文显示。

## 实现摘要

- `MapDisplayAdapter`：
  - 为 `RegionInspectorState` 增加 selected hex / region 的动态方面和防区显示名。
  - 为 `UnitInspectorStrategicState` 增加州府、动态方面、防区和粮源显示名。
  - 将州府目标状态改为场景感知：唐宋路径显示“无目标”或“某政权控制”，legacy 保持英文状态。
- `UnitInspectorView`：
  - 唐宋路径优先显示运行态州府、动态方面和防区名称，缺名时显示“未知州府 / 未命名方面 / 未命名防区”。
  - 唐宋战线摘要显示为“相关战线 N 条”，避免直接展示 `FrontLineId.rawValue`。
  - 唐宋粮道摘要使用州府名或“补给源”兜底，避免直接展示 supply source id。
- `RegionInspectorView`：
  - selected hex 与州府的动态方面 / 防区优先显示运行态名称。
  - 缺名时使用“未命名方面 / 未命名防区”，legacy 路径仍可显示 raw id。

## 并发子 Agent 结论

- inspector explorer 只读确认高优先级残留集中在 `UnitInspectorView` 的 region/theater/frontZone raw id、frontLineSummary、supplyRouteSummary，以及 `RegionInspectorView` 的 selected hex / region theater/frontZone raw id 和 objectiveStatus。
- map/objective explorer 只读确认 `MapDisplayAdapter.inspectorState` 仍生成英文 `None` / `controlled`，并指出 `AppContainer.selectionMessage` 的唐宋路径已由 v5.8i 收口，非唐宋 raw id 不属于本轮目标。
- 两个子 Agent 均未编辑文件、未运行测试/构建、未触碰 git。

## 验证口径

- 本机按用户要求不运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 本轮只做只读差异审查与残留搜索。
- 提交推送 `origin/main` 后，由 GitHub Actions `WWIIHexV0 CI Results` 云端运行静态检查和 Xcode build，并下载未加密 artifact 核对 manifest、JUnit、static checks、xcodebuild log 和 failure summary。

## 风险与后续

- raw id 仍保留在 state 字段中，供 legacy 显示、排序、调试和规则链路使用；唐宋路径只改变玩家可见显示桥。
- 完整 accessibility、截图/布局验收、其他面板英文/raw id 残留和正式 RC 清单仍需后续 v5.8/v5.9 小切片。
