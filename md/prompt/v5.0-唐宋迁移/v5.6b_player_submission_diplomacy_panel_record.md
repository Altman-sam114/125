# v5.6b 玩家招抚入口与外交面板只读记录

本文记录 v5.6 的第二个小切片：在 v5.6a 已有 `Command.proposeSubmission` 规则合同的基础上，给玩家提供可见的“招抚”入口，并让外交面板只读展示天命与归附记录。它仍不是完整外交系统、完整纳土交割、治理政策或唐宋胜利规则。

## 1. 目标

- `CommandPanelView` 新增“招抚 / Propose Submission”按钮。
- `AppContainer.proposeSubmissionSelected()` 把玩家操作转换为 `Command.proposeSubmission` 并交给既有 `submit -> RuleEngine`。
- 招抚目标优先使用当前选中的外国首府；若当前选中 region 不是合法目标，则自动扫描当前可招抚首府，开局可显示吴越。
- `DiplomacyPanelView` 只读展示 `MandateState` 天命分数和最近 `PacificationRecord`。
- 唐宋场景下外交面板显示“外交 / 天命 / 诸国 / 集团 / 关系 / 归附记录”等读法。

## 2. 非目标

- 不让 UI 直接写 `GameState`、`DiplomacyState` 或 `MandateState`。
- 不改变 `Command.proposeSubmission` 的规则语义，不新增距离、范围或逐州府国家归属规则。
- 不把 `TheaterDirectiveEnvelope.pacificationTargets` 自动编译成归附命令。
- 不交割 hex / region controller，不转换或删除部队，不刷新 theater/front/deploy。
- 不改变 `TurnOrderState.relations` 的 `.allies/.germany` 全局战争关系。
- 不实现 `GovernancePolicy`、民心、治安、税粮、叛乱或唐宋专用胜利规则。

## 3. UI 到规则链路

玩家按钮走 direct `Command -> RuleEngine` 路径；`WarCommandExecutor` 仍只负责 `ZoneDirective` 降级，不参与本轮玩家招抚按钮。

```text
CommandPanelView
  -> AppContainer.proposeSubmissionSelected()
  -> Command.proposeSubmission(negotiatorId:targetCountryId:targetRegionIds:)
  -> AppContainer.submit(_:)
  -> RuleEngine.execute
  -> CommandValidator.validateProposeSubmission
  -> CommandExecutor.executeProposeSubmission
  -> DiplomacyState / MandateState / diplomacy event log
  -> DiplomacyPanelView read-only display
```

按钮启用条件由 `AppContainer.selectedSubmissionTarget` 派生：

- 必须有当前可行动玩家军队。
- 目标必须能从 `CountryProfile.capitalRegionId` 映射到外国首府 region。
- 候选命令必须通过 `CommandValidator` 预校验。
- 若玩家已选中合法外国首府，优先用该目标；否则按 `warSupport` 从低到高扫描合法首府。

当前没有完整 region -> country 归属表，因此非首府 region 不会推断国家。若玩家选中非首府或非法 region，按钮仍可能指向自动扫描出的合法首府目标；按钮文案会显示实际目标国家名。辽边境压力虽然有 `capitalRegionId`，但当前 hostile 且不能形成围城压力，开局不会成为合法招抚目标。

## 6. 遗留风险

- 当前玩家招抚入口仍只支持首府推断，不支持点击吴越其他州府推断吴越。
- 选中非首府或非法 region 时，按钮可能 fallback 到自动扫描出的最低 warSupport 合法首府；后续若要严格“点哪谈哪”，需要移除自动 fallback 或补更明确的目标选择 UI。
- 招抚命令没有距离限制；这更像朝廷外交提议，而不是战场近距谈判。若后续要绑定兵临城下，应在 `CommandValidator` 增加距离或压力条件。
- 招抚成功只写关系、记录和天命，不代表纳土、控制权交割、部队转换或 tactical neutral。
- 天命仍未影响 `VictoryRules`，完整统一评价留给后续 v5.6/v5.7 切片。
