# v5.8aa MapEditor 画布 value 与底图控件硬化记录

## 目标

继续收口 MapEditor 的读屏语义和玩家可见说明，让地图编辑画布、底图导入与底图参数控件更清楚，并减少默认编辑器路径中的 ASCII 快捷键/轴字母表达。

## 实现范围

- `MapEditorView` / `MapEditorSpriteView`：
  - 地图编辑画布补充 accessibility value，读出当前模式、编辑状态、地块/州府/方面/军队数量、选中地块、待加入/待部署数量和底图状态。
  - 画布 accessibility hint 改成“地图设置”，不再把底图参数作为读屏主提示。
  - 编辑动作帮助文案去掉 `Option+左键`、`N`、`M` 快捷键表达，改为平台中性说明。
  - 底图导入、移除、缩放、横向/纵向偏移和应用参数控件补充 accessibility label/value/hint。
  - 底图偏移输入框去掉可见 `X/Y` 轴字母。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 并发子 Agent

- MapEditor 只读扫描指出画布缺少 accessibility value，帮助文案仍有 `Option+左键`、`N`、`M`，底图控件需要更明确上下文，偏移输入框仍有 `X/Y` 轴字母。
- 文档扫描用于确认 v5.8z 后续记录位置；本轮按 v5.8 系列现有 README、plan、flow、update_log 口径追加 v5.8aa。
- 本轮只纳入 MapEditor SwiftUI 可读语义和说明文字，不修改 SpriteKit 编辑行为、快捷键处理或导出资源结构。

## 禁止项

- 不改变 `MapEditorDocument`、导出 JSON schema、资源桥路径、`Faction.allies/germany`、`RegionId`、`TheaterId`、主游戏规则或 Codable raw 值。
- 不改变 `MapEditorCanvasScene` 画布绘制、鼠标事件、滚轮/平移/缩放逻辑、`handleShortcut(_:)` 或 macOS 键盘快捷键绑定。
- 不做完整 VoiceOver 实机验收、截图验收、macOS 编辑器布局验收、Dynamic Type 视觉验收或发布级 UI 验收。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成发布级 UI 验收完成。

## 遗留风险

- MapEditor 真实 VoiceOver focus order、画布逐地块可访问路径、截图布局和 macOS 编辑器发布级体验仍未验收。
- 画布内“补”与军队短字标仍作为地图编辑符号保留；若后续需要替换为正式图标系统，应单独规划视觉/符号切片。
- 完整 v5.8 RC 玩家可见残留清单、资源授权、性能和 v5.9 发布验收仍待后续。
