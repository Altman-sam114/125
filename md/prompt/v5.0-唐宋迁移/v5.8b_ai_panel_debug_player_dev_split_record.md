# v5.8b AI 面板玩家态/开发态分层记录

本文记录唐宋 v5.8b 小切片：把 AI 面板中的 diagnostics、错误原文和 raw JSON 从默认玩家态拆到折叠调试区。该切片只做 UI 展示层分层，不改变 AI 记录、命令、规则或 JSON schema。

## 目标

- 让唐宋默认 AI 面板优先显示军议摘要、方面军令和命令执行/拒绝摘要。
- 避免默认铺开英文 diagnostics、schema key、内部 id、validation rawValue 或规则错误原文。
- 保留开发态可追溯性：diagnostics、errors 和 raw JSON 仍可通过折叠调试区查看。
- 找不到运行态州府/防区名称时，用“未命名州府 / 未命名方面”兜底，不把 raw id 当玩家文案。
- 保持非唐宋 legacy 路径原显示不变。

## 非目标

- 不改 `AgentDecisionRecord`、`WarDirectiveRecord`、`TheaterDirectiveEnvelope`、`Command`、`ZoneDirective` 或 raw JSON schema。
- 不改 `TheaterDirectiveDecoder`、`TheaterDirectiveCompiler`、`WarCommandExecutor`、`RuleEngine`、`CommandValidator`、`CommandExecutor` 或 AI 决策算法。
- 不接真实 LLM，不改 provider，不改变 fallback 策略。
- 不处理外交面板、EventLog 原始日志、MapEditor 英文错误、README legacy 段落或全项目玩家可见残留。
- 不做截图、iPhone/iPad 布局、VoiceOver、Xcode build/test 或本地轻量检查。

## 实现说明

```text
AgentPanelView 唐宋路径
  玩家态：
    意图 -> theaterDirectiveSummary.summary / strategicIntent / 已形成方面军令
    战况 -> 本地化战况兜底
    命令结果 -> 已执行 / 规则拒绝 / 映射失败 / 未执行
    方面军令 -> 唐宋战术、州府名、防区名

  开发态折叠：
    军令诊断 -> WarDirectiveRecord.diagnostics
    军议未成 -> AgentDecisionRecord.errors
    军议原文 -> AgentDecisionRecord.rawJSON
```

唐宋显示桥补充：

- `resultLine(_:)` 不直接显示 `CommandResultSummary.message` 或 `errors.joined(...)`，改为短摘要。
- `rawJSONSection` 在唐宋路径下使用 `DisclosureGroup("军议原文")`；无 raw JSON 时显示“暂无军议原文”。
- diagnostics 和 errors 使用折叠区展示原文，避免默认玩家态暴露英文或内部 id。
- `intentDisplayText` 优先读取 `theaterDirectiveSummary.summary / strategicIntent`，避免显示 `marshal directives`、`json` 等 fallback 字符串。
- region / zone fallback 在唐宋路径下显示“未命名州府 / 未命名方面”。

## 并发子 Agent 结果

本轮继续使用并发子 Agent，但保持只读审计：

- 代码审计子 Agent 指出 raw JSON、resultLine、diagnostics、record.errors、parsedIntent/contextSummary 和 region/zone fallback 仍可能暴露英文、schema key 或 raw id；主线程已按这些点收口。
- 文档审计子 Agent 给出必须同步的 README、plan、flow、mermaid、update_log 和新阶段记录清单；主线程按清单更新，不改 `AGENTS.md` 或 `md/test/test.md`。

主线程没有让子 Agent 编辑文件，没有让多个 Agent 同时改同一 public API 或同一文件。

## 验证口径

用户明确要求“不在本地测试，都去云端测试”，因此本轮不运行本地测试、build、Swift parse、Markdown 检查、`git diff --check` 或其他本地轻量检查。

推送到 `origin/main` 后，由 GitHub Actions `WWIIHexV0 CI Results` 做云端静态检查和 Xcode build；Agent C / 主线程再下载未加密 artifact，核对 manifest、JUnit、`ci-failure-summary.md`、`static-checks.log` 和 `xcodebuild.log`。

## 遗留风险

- diagnostics、errors 和 raw JSON 仍在开发态折叠区可见，这是调试可追溯性保留，不是 schema 迁移。
- 外交面板、EventLog 原始日志、MapEditor 英文错误、README legacy 段落和全局 accessibility 文案仍待 v5.8 RC 审计。
- 未做截图、布局、VoiceOver 或真机/模拟器运行验收。
