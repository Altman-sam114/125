# v5.8af 将领军令面板指标与列表读屏硬化记录

## 目标

继续收口唐宋默认主路径中的将领军令面板，减少忠诚、军心、亲征干预等裸数字，并让所属军队和已拟军令在读屏时保留完整上下文。

## 实现范围

- `GeneralCommandPanelView`：
  - 忠诚/军心指标补充合并 accessibility label/value，唐宋路径读作“数值，满百”。
  - 亲征干预显示为“N 次”，并补次数读屏值。
  - 所属军队行补读屏 label/value，读出军队名、兵力和满额。
  - 已拟军令行补读屏 label/value，读出“已拟军令：进攻/固守”和目标名。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
  - 将 v5.8ae 的云端通过结果补写到 `update_log.md`，保留 v5.9 未开始边界。

## 并发子 Agent

- 主游戏 UI 只读扫描指出 `GeneralCommandPanelView` 的指标、所属军队和已拟军令仍缺少完整读屏上下文。
- 文档/战报扫描指出 `EventLogView` 战报行整行读屏也有价值；本轮选择已经定位且不与 v5.8z 重复的将领军令面板，战报行保留为后续小切片候选。

## 禁止项

- 不改变 `GeneralData`、`GeneralAssignment`、`FrontZone`、`Division`、`PlayerPlannedOperation` 或 Codable schema。
- 不改变 `onHoldLine` / `onAttackRegion` 行为，不改变 `ZoneDirective -> WarCommandExecutor -> RuleEngine` 命令链路。
- 不改变将领分配、AI 决策、规则校验、hex/region/theater/front/deploy 权威边界。
- 不声明完整 VoiceOver 实机验收、截图布局验收、全项目玩家可见残留清零或发布级 UI 完成。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成完整全项目写入端本地化或发布级 UI 验收完成。

## 遗留风险

- `EventLogView` 战报行整行读屏、低频面板 raw id / 英文 fallback 和其它短数字读法仍需继续按 v5.8/v5.9 小切片扫描。
- 完整 VoiceOver 实机、截图布局、全项目玩家可见残留扫描、真 LLM 输出本地化和发布级 UI 验收仍未完成。
