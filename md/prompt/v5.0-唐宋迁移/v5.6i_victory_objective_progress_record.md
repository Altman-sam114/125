# v5.6i 胜利目标进度只读显示记录

本文记录唐宋 v5.6i 小切片：在 v5.6g 已能从场景 JSON 读取胜利条件、v5.6h 已能显示胜负原因的基础上，把胜利目标当前进度以只读方式显示到 HUD 与战报面板。

## 1. 目标

- 新增纯只读 `VictoryObjectiveProgress` 快照，用于描述胜利条件的当前达成度。
- `VictoryRules.objectiveProgress(in:)` 读取 `GameState.victoryConditions`、objective 所在 hex controller 和 `MandateState`，生成州府控制、天命阈值和回合门槛进度。
- HUD 显示主要统一条件的州府进度与天命进度。
- 战报面板显示主要胜利目标的标题、当前门槛、目标州府和达成状态。

## 2. 非目标

- 不新增胜利条件，不改变 `VictoryRules.updateVictoryState(in:)` 判定语义。
- 不调用会写 `VictoryState` 的胜负更新逻辑来做展示计算。
- 不向 `eventLog` 写派生进度事件。
- 不做完整胜利面板、结算页、评分档位、治理评价、单国胜负或统一结算。
- 不从 region / theater / diplomacy 推断 objective 控制权；仍以 objective 坐标所在 hex controller 为准。

## 3. 实现链路

```text
GameState.victoryConditions
  -> VictoryRules.objectiveProgress(in:)
  -> MapState.controllerOfObjective(id:)
  -> MandateState.legitimacy(for:)
  -> HUDView 只读统一/天命进度
  -> EventLogView 只读胜利目标摘要
```

旧场景或旧存档缺少 `victoryConditions` 时，进度显示使用 v5.6d 关键州府与天命阈值 fallback，和当前唐宋胜负 fallback 保持同一口径。

## 5. 并发审计

本轮按用户要求继续使用并发子 Agent：

- UI 子 agent 只读审查 HUD、战报、RootGameView 与外交面板，确认最小接入点是 HUD 和现有战报摘要区域，不新增 tab，不新增工程文件。
- 规则/数据子 agent 只读审查 `VictoryRules`、`VictoryConditionDefinition`、`GameState` 和 `MapState`，确认进度计算必须是纯函数，读取 objective id、count、turn / turns、`mandateThreshold` 和 `MandateState`，不得调用 `updateVictoryState(in:)`。

主线程采用公共只读查询方案，避免 UI 复制规则私有逻辑。
