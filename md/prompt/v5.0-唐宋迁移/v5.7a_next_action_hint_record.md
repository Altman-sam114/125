# v5.7a 首屏下一步只读提示记录

本文记录唐宋 v5.7a 小切片：在 v5.6 已能显示胜负原因和胜利目标进度的基础上，把新玩家最需要的“下一步做什么”以只读方式放到 HUD。

## 1. 目标

- 在唐宋场景的首屏 HUD 中显示一句“下一步”提示。
- 提示按胜负状态、观战模式、当前回合权限、选中军队、围城/招降/招抚/解围/修城候选派生。
- 帮助玩家发现既有命令入口：选军、行军、围城、招降、招抚、解围、修城、结束回合和查看战报。

## 2. 非目标

- 不做完整教程系统、说明页、landing page 或新手任务链。
- 不新增命令，不改变 `CommandValidator`、`RuleEngine` 或任何规则判定。
- 不把提示写入 `GameState`、`eventLog`、`VictoryState`、hex、region、theater、front 或 diplomacy。
- 不让提示替代命令按钮的真实合法性检查。
- 不做截图、模拟器、Xcode build/test 或完整布局验收。

## 3. 实现链路

```text
RootGameView.nextActionHint
  -> 读取 GameState / observerModeEnabled / selectedDivision
  -> 读取 selectedDemandSurrenderTargetName / selectedBesiegeTargetName
  -> 读取 selectedSubmissionTargetName / selectedRelieveSiegeTargetName / selectedRepairFortificationTargetName
  -> HUDView(nextActionHint:)
  -> 唐宋场景只读显示“下一步”
```

真实行动仍由玩家点击既有按钮后提交底层 `Command`，再经 `RuleEngine -> CommandValidator -> CommandExecutor` 执行。
