# v5.8a AI 面板默认主路径残留硬化记录

本文记录唐宋 v5.8a 小切片：对 AI 面板默认唐宋主路径做首轮残留硬化。该切片只做玩家可见显示桥，不改变 AI 记录 schema、directive 编译、命令执行或规则权威。

## 目标

- 减少 `AgentPanelView` 在唐宋默认主路径中直接显示 Guderian、MockAI、global、move/attack/hold/resupply、raw region id、raw front zone id 等残留。
- 让 AI 面板的主事、来源、君主、将令、防区、目标州府、命令标题和国策姿态使用唐宋读法。
- 让 `RootGameView` 向 AI 面板传入运行态州府与防区名称查找，优先显示当前数据中的州府名和方面防区名。
- 保持非唐宋 legacy 路径原显示不变。

## 非目标

- 不改 `AgentDecisionRecord`、`WarDirectiveRecord`、`TheaterDirectiveEnvelope`、`Command`、`ZoneDirective` 或 Codable raw schema。
- 不改 `TheaterDirectiveCompiler`、`WarCommandExecutor`、`RuleEngine`、`CommandValidator`、`CommandExecutor` 或 AI 决策算法。
- 不隐藏或重构 raw JSON 调试区，不处理全部 diagnostics、错误消息和 schema key。
- 不处理外交面板、EventLog 原始日志、MapEditor 英文错误、README legacy 段落或全项目玩家可见残留。
- 不做截图、iPhone/iPad 布局、VoiceOver、Xcode build/test 或本地轻量检查。

## 实现说明

```text
RootGameView
  -> AgentPanelView(
       isTangSongScenario: gameState.isTangSongScenario,
       regionDisplayName: gameState.map.regions[id]?.name ?? id.rawValue,
       zoneDisplayName: gameState.warDeploymentState.frontZones[id]?.name ?? id.rawValue
     )

AgentPanelView
  -> agent/provider/ruler/commander display bridge
  -> zone / region display bridge
  -> command result title bridge
  -> ruler posture and legacy order type display bridge
```

唐宋显示桥：

- agent fallback：宋枢密院、割据行营、君主诏令、方面主将、军议主事。
- provider fallback：确定性军议、本地军议、军议来源。
- nil zone：全局军令。
- region / zone id：优先读取运行态州府名和方面防区名。
- legacy order type：行军、进攻、固守、整补。
- ruler posture：进取、固守、维系诸国、安定边面。
- command result title：唐宋场景下只显示行军、进攻、围城、修城、解围、招降、招抚、招抚候选、固守、准退、休整、军备、结束回合等动作名，避免把括号里的单位 id、国家 id 或 region id 当成玩家标题。

## 并发子 Agent 结果

本轮按用户要求继续使用并发子 Agent 加速，但保持只读审查：

- 上一轮只读子 Agent 已定位 AI 面板 raw id、provider、ruler posture、zone、region targets、legacy order type、commander id、diagnostics、error 和 raw JSON 等残留点，并建议文档同步范围。
- 本轮只读子 Agent 审查当前 Swift 改动，未发现明显签名或执行层误改风险；提醒 `commandDisplayName` 仍可能泄露括号内 raw id，主线程已改成唐宋命令标题桥；同时提醒 raw JSON 调试区仍是 schema 形态，本轮记录为后续 RC 风险。

主线程没有让子 Agent 编辑文件，没有让多个 Agent 同时改同一 public API 或同一文件。

## 验证口径

用户明确要求“不在本地测试，都去云端测试”，因此本轮不运行本地测试、build、Swift parse、Markdown 检查、`git diff --check` 或其他本地轻量检查。

推送到 `origin/main` 后，由 GitHub Actions `WWIIHexV0 CI Results` 做云端静态检查和 Xcode build；Agent C / 主线程再下载未加密 artifact，核对 manifest、JUnit、`ci-failure-summary.md`、`static-checks.log` 和 `xcodebuild.log`。

## 遗留风险

- AI 面板 diagnostics 仍可能显示英文或内部管线信息。
- AI 面板错误消息仍可能显示英文校验、mapping 或规则错误原文。
- raw JSON 调试区仍显示 schema key 和内部 id；本轮只把默认占位值改为唐宋读法，没有做玩家态/开发态分层。
- 外交面板、EventLog 原始日志、MapEditor 英文错误、README legacy 段落和全局 accessibility 文案仍待 v5.8 RC 审计。
- 未做截图、布局、VoiceOver 或真机/模拟器运行验收。
