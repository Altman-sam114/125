# v5.8g 主游戏默认启动 fallback 硬化记录

## 目标

- 延续 v5.8 发布候选硬化，让主游戏默认启动只加载唐宋 960 三件套：`tangsong_jianlong_960_scenario`、`tangsong_jianlong_960_regions`、`tangsong_unit_templates`。
- 唐宋默认资源缺失、JSON 解码失败、引用校验失败或单位模板缺失时，不再静默回退 legacy 阿登或 hardcoded `GameState.initial()`。
- 保留阿登数据、显式 legacy 加载入口和历史回归参考，不删除旧资源或旧测试夹具。

## 非目标

- 不删除 `ardennes_v0_scenario.json`、`ardennes_v02_regions.json`、`unit_templates.json`、Guderian / Bastogne 历史夹具或旧测试。
- 不重命名 `Faction.allies/germany`、`GamePhase.germanAI/alliedPlayer`、`Division`、`ComponentType` 或任何 Codable schema。
- 不改规则系统、命令管线、AI pipeline、MapEditor JSON schema 或 hex / region / theater / front / deploy 权威边界。
- 不处理本轮 UI 子 Agent 扫出的将领 catalog、命令面板 raw validation、inspector raw id、accessibility 和截图/布局问题。
- 不做本机测试、build、Swift parse、Markdown 检查或 `git diff --check`。

## 实现说明

- `DataLoader.loadInitialGameState()` 改为严格加载唐宋默认资源；失败时 `assertionFailure` 并返回唐宋错误态，错误态使用空地图、宋/割据 turn order、唐宋天命初值和中文事件日志。
- 原默认链路中的阿登 JSON fallback 与 hardcoded `GameState.initial()` fallback 移到显式 `loadLegacyArdennesGameState()`，供历史回归或手动 legacy 调用保留。
- `makeDivisions` 不再把任意未知 `templateId` 静默降级成 infantry。只有已知 legacy 模板 id 可以走 `legacyFallbackComponents`；唐宋模板错名会抛 `DataLoaderError.validationFailed`。

## 并发子 Agent 结果

- DataLoader 子 Agent 只读确认 `loadInitialGameState()` 是主游戏 `bootstrap()` 与 `resetGame()` 的真实默认路径，唐宋加载失败会被 `try?` 吞掉并静默进入阿登；本轮已处理。
- 同一子 Agent 指出 `makeDivisions` 未知模板会 fallback infantry；本轮已改为未知模板报错。
- UI/accessibility 子 Agent 只读指出 legacy `generals.json`、命令面板 raw validation、inspector raw id、EventLog relatedRecordId 和部分 VoiceOver 文案仍是后续 v5.8/v5.9 高价值残留。本轮未混入这些 UI 改动。

## 验证口径

- 按用户当前要求，本轮不在本机运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 本轮完成后直接 commit 并 push 到 `origin/main`，触发 GitHub Actions `WWIIHexV0 CI Results`。
- Agent C / 主线程验收必须下载最新 `origin/main` commit 对应的未加密 artifact，核对 `ci-artifact-manifest.json` 的 `branch`、`commitSha`、`runId`、`runAttempt`，并阅读 `junit.xml`、`static-checks.log`、`xcodebuild.log` 和 `ci-failure-summary.md`。

## 遗留风险

- 唐宋默认资源失败时目前返回空地图错误态，而不是专门的错误 UI；后续可设计发布态错误面板。
- `loadLegacyArdennesGameState()` 只保留给显式 legacy 调用；历史测试若继续把 `loadInitialGameState()` 当阿登入口，需要改用显式 legacy helper。
- 唐宋将领面板仍可能读取 legacy `generals.json` 的英文人物与技能；命令面板、inspector 和部分 accessibility 仍有后续残留收口空间。
