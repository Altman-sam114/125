# v5.7h 亲征势力/观战轻量入口记录

本文记录唐宋 v5.7h 小切片：在 v5.7f 已有“指挥 / 模式”只读身份包装、v5.7g 已让下一步提示读取高亮数量后，为唐宋主界面补上可操作的“亲征势力 / 观战”轻量入口。

## 1. 目标

- 唐宋场景主界面提供“亲征”分段选择，允许玩家在现有 legacy `.allies / .germany` 桥上切换当前亲征阵营。
- 唐宋 turn order 初始化读取场景 JSON 的 `playerFaction` / `aiFaction`，让默认亲征与 AI 控制口径来自数据而不是硬编码。
- 保留既有“观战”入口；观战时玩家命令停用，并可继续让 AI 推进当前可运行回合。
- 切换亲征阵营后，HUD 指挥身份、军令可用性、单位选择、高亮和下一步提示都读取新的 `playerFaction`。
- 同步 `TurnOrderState.playerControlledPowerIds` 和 legacy profile controlMode，避免 UI 亲征阵营与回合控制口径分叉。

## 2. 非目标

- 不新增 `Faction` case。
- 不新增吴越、南唐、后蜀、北汉等真实多政权选择器。
- 不新增 `GameState` schema、存档槽、持久化用户配置或开局向导页。
- 不改变 `Command`、`CommandValidator`、`CommandExecutor`、`RuleEngine`、`TurnManager` 或 `WarCommandExecutor` 的执行语义。
- 不改变胜利条件、外交关系、hex / region / theater / front / deploy 权威边界。
- 不做截图、布局、VoiceOver、模拟器或本机 Xcode 验收。

## 3. 实现链路

```text
RootGameView
  -> 唐宋场景显示“亲征” Picker + “观战” Toggle
  -> Picker 调用 AppContainer.setPlayerFaction(_:)

DataLoader.initialTurnOrderState
  -> 读取 scenario.playerFaction / aiFaction
  -> 初始化 PowerProfile.controlMode
  -> 初始化 TurnOrderState.playerControlledPowerIds

AppContainer.setPlayerFaction(_:)
  -> 更新运行时 playerFaction
  -> configuredState 同步 TurnOrderState.playerControlledPowerIds
  -> 同步带 legacy bridge 的 PowerProfile.controlMode
  -> 清空 selectedUnitId / movementHighlights / attackHighlights
  -> refreshSelectionAfterStateChange()
  -> runAIIfNeeded()

HUDView / CommandPanelView / EconomyPanelView / nextActionHint
  -> 继续读取 AppContainer.playerFaction / observerModeEnabled
  -> 不提交命令，不直接改规则结果
```

真实移动、攻击、围城、招抚、结束回合仍由现有 UI 入口提交 `Command` 或 `ZoneDirective`，再经 `RuleEngine` / `WarCommandExecutor` 执行。
