# v5.8e MapEditor 默认路径读法硬化记录

## 目标

- 延续 v5.8 发布候选硬化，减少唐宋地图编辑器默认路径中的阿登 fallback、英文错误、英文导出注记、raw id 和二战兵种短标暴露。
- 让“读取唐宋默认资源”只读取唐宋 960 默认资源，缺失时显式报错，不静默回退 legacy 阿登资源。
- 保持 MapEditor 的 JSON schema、底层 raw id、`Faction` 兼容桥和导出结构不变，只处理默认工具路径的玩家可见读法。

## 非目标

- 不改变主游戏 `DataLoader.loadInitialGameState()` 的 legacy 阿登 fallback；该策略属于主游戏启动容错，后续单独切片评估。
- 不改 `ScenarioDefinition`、`RegionDataSet`、`MapEditorDocument` Codable schema 或 JSON key。
- 不改 `Faction.allies/germany`、`RegionId`、`TheaterId`、`Division` 等兼容类型名。
- 不新增单位模板/人物 JSON 编辑能力，不改 MapEditor 测试夹具，不做运行时 UI 点击、截图或布局验收。

## 实现说明

- `MapEditorGameResourceBridge.defaultResourceURLs()` 移除编辑器默认读取的 legacy 阿登 fallback；唐宋资源缺失时返回中文错误。
- `MapEditorDocument.new` 默认草案改为 `tangsong_mapeditor` / “唐宋地图草案”。
- `MapEditorExporter` 的导出错误、补给点 fallback、dataNotes 和 `RegionDataSet.displayName` 改为中文唐宋口径。
- `MapEditorViewModel` 的新建州府/方面、自动城市名和导出状态不再默认暴露 raw id 或完整本机路径。
- `MapEditorView` 的州府/方面选择器、信息面板和资源路径说明默认显示中文名称/默认文件名，不直接铺开 raw id 和绝对目录。
- `MapEditorCanvasScene` 的单位短标从 `ARM/ART/MOT/INF` 改为禁、骑、弩、械、守、州、军等唐宋读法。

## 并发子 Agent 结果

- MapEditor 子 Agent 只读确认高优先级残留集中在编辑器资源 fallback、Picker/信息面板 raw id、英文导出错误、默认城市名、补给点 fallback、英文导出注记、默认草案名和二战兵种短标；本轮已覆盖这些默认路径。
- 文档子 Agent 只读指出 `md/flow/04_mapeditor_to_game_data.mermaid` 仍画阿登默认写回，本轮已同步为唐宋默认文件；同时指出 README、plan、flow 顶部和 update_log 仍有更大范围历史定位残留，保留给后续 docs-only 切片。

## 验证口径

- 按用户当前要求，本轮不在本机运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 本轮完成后直接 commit 并 push 到 `origin/main`，触发 GitHub Actions `WWIIHexV0 CI Results`。
- Agent C 验收必须下载最新 `origin/main` commit 对应的未加密 artifact，核对 `ci-artifact-manifest.json` 的 `branch`、`commitSha`、`runId`、`runAttempt`，并阅读 `junit.xml`、`static-checks.log`、`xcodebuild.log` 和 `ci-failure-summary.md`。

## 遗留风险

- 主游戏启动路径仍可能在唐宋默认资源缺失时回退 legacy 阿登；这不是 MapEditor 行为，后续需单独决定是否改为发布态显式错误。
- README、`md/plan/plan.md`、`md/flow/flow.md` 顶部标题和部分历史段落仍以 WWIIHexV0/v0.x/阿登为主，建议下一轮做 docs-only 大纲收口。
- MapEditor 测试夹具仍有 German / Allied / Panzer 命名，未影响默认 UI，但后续维护测试时应同步唐宋化。
