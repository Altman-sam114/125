# v5.6c AI 招抚候选到规则命令安全桥记录

本文记录唐宋 v5.6c 小切片：让 AI 元帅 envelope 的 `pacificationTargets` 不再只停留在只读解释摘要，而是在安全边界内尝试生成 `Command.proposeSubmission`。该切片仍不是完整外交纳土、治理或胜利规则。

## 1. 目标

- 在唐宋场景下读取 `TheaterDirectiveEnvelope.pacificationTargets`。
- 在战争 `ZoneDirective` 执行后、`.endTurn` 前，把首府招抚候选尝试转换为辅助 `Command.proposeSubmission`。
- 让辅助命令继续通过 `RuleEngine -> CommandValidator -> CommandExecutor` 决定成功、规则拒绝或跳过。
- 将 AI 招抚尝试写入 `AgentDecisionRecord.commandResults`，便于 AI 面板和后续 artifact 审计。
- 每个 AI 回合最多允许 1 个成功招抚提议，避免单轮外交副作用过大。

## 2. 非目标

- 不改变 `Command.proposeSubmission` 的规则语义。
- 不改 `TheaterDirectiveCompiler`，不让 `WarCommandExecutor` 承担外交语义。
- 不交割 hex / region controller，不转换、删除或收编部队。
- 不改变 `TurnOrderState.relations` 的 `.allies/.germany` 全局战争关系。
- 不实现 `GovernancePolicy`、民心、治安、税粮、叛乱或唐宋胜利规则。
- 不让 `mandateIntent`、`courtPolicy` 或 `supplyPriorities` 落到规则执行。
- 不接真实 LLM、网络、API key 或外部模型。

## 3. 执行链路

```text
MarshalAgent.resolve
  -> TheaterDirectiveEnvelope.pacificationTargets
  -> TurnManager.runMarshalDirectiveTurn
  -> TurnManager.executeDirectiveEnvelope
  -> WarCommandExecutor 执行战争 ZoneDirective
  -> TurnManager.executePacificationTargets
  -> Command.proposeSubmission
  -> commandHandler.execute
  -> RuleEngine
  -> CommandValidator / CommandExecutor
  -> AgentDecisionRecord.commandResults
  -> Command.endTurn
```

`executePacificationTargets` 只在 `state.isTangSongScenario` 为 true 且候选非空时运行。它会去重候选 region，按 `CountryProfile.capitalRegionId` 反查外国目标国家，再选择当前 faction 的未行动、未撤退、可行动军队作为谈判军队。候选成功、被 validator 拒绝、找不到国家或找不到谈判军队，都会成为可审计结果；常规拒绝不写入顶层 `errors`，避免把规则层预期拒绝误报成 AI 执行异常。

## 6. 并发子 Agent 整合

本切片使用并发子 Agent 做只读审计：一个检查 Swift diff 是否仍保持 `Command -> RuleEngine` 边界，一个检查文档中关于 `pacificationTargets` 的旧口径。主线程负责最终整合和落地。

并发整合重点：

- 多个子 Agent 不改同一文件；最终改动由主线程统一合并。
- `Command`、`CommandValidator`、`CommandExecutor`、`RuleEngine` 语义不分叉。
- 文档统一写成“AI 可尝试发起规则校验的招抚提议”，不写成自动归附、自动停战或自动控制权交割。

## 7. 遗留风险

- 默认 AI faction 仍桥接到 legacy `.germany` / `.allies`；目标国家是否能成功归附完全由现有 validator 条件决定，常见结果可能是规则拒绝或跳过。
- 目标国家仍通过首府 region 反查；非首府州府不能单独推断国家。
- `DiplomacyState` 与 `TurnOrderState.relations` 仍是两套关系来源；本切片不解决 tactical neutral 或单国战争关系。
- 天命仍不影响 `VictoryRules`，归附记录也不交割地图控制权。
