# v5.7l 将领指挥/档案面板唐宋读法记录

本文记录唐宋 v5.7l 小切片：在 v5.7k 补齐军队/州府检查面板后，继续补将领指挥与将领档案面板的唐宋显示桥，减少默认主路径里的英文将领面板和 legacy 势力名残留。

## 1. 目标

- 唐宋场景下，`GeneralCommandPanelView` 显示将领军令、方面防区、查看档案、所属军队、目标州府、固守防线、进攻州府和已拟军令等读法。
- 唐宋场景下，`GeneralProfileView` 显示将领档案、履历、用兵、所辖方面、朝廷关系、忠诚、军心、亲征干预、特长和辖下军队等读法。
- 将领所属政权通过 `GameState.displayName(for:)` 显示，避免唐宋主路径直接露出 `Germany` / `Allies`。
- 将领用兵风格在唐宋路径显示为锐进、持重、谨慎；计划摘要把 `attack/defend` 显示为进攻/固守。

## 2. 非目标

- 不改 `GeneralData`、`GeneralAssignment`、`FrontZone`、`PlayerPlannedOperation` 或存档 Codable schema。
- 不改将领分配、AI 决策、`ZoneDirective`、`WarCommandExecutor`、`Command`、`CommandValidator`、`RuleEngine` 或胜负规则。
- 不新增真实多政权选择器、历史人物数值系统、技能完整唐宋词典或发布级布局验收。
- 不跑本机测试、构建、模拟器或截图验收；本轮按用户要求只走云端验证。

## 3. 实现链路

```text
RootGameView
  -> GeneralCommandPanelView(isTangSongScenario)
  -> GeneralProfileView(isTangSongScenario, factionDisplayName)
  -> 只读显示唐宋将领面板读法
  -> 不提交命令，不写 GameState
```

`factionDisplayName` 由 `RootGameView` 传入 `container.gameState.displayName(for:)`。将领操作按钮仍调用既有 `orderSelectedGeneralHoldLine` / `orderSelectedGeneralAttackRegion`，后续仍经 `ZoneDirective -> WarCommandExecutor -> RuleEngine`，本轮不改变执行链路。
