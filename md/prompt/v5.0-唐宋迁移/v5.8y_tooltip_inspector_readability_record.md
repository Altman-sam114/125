# v5.8y tooltip 与检查面板读法硬化记录

## 目标

继续收口主游戏默认路径的 tooltip 与检查面板可读语义，让读屏和可见文本少暴露内部缩写、类型名或不完整状态。

## 实现范围

- `UnitTooltipView`：
  - accessibility label 保留军队名和兵种。
  - accessibility value 补充兵力、补给、退却和本回合状态。
  - 英文 low supply tooltip 从 `Low` 改为 `Low Supply`。
- `UnitInspectorView`：
  - 英文 `FrontZone` / `FrontLine` / `Deploy` 标签改为 `Front Zone` / `Front Lines` / `Deployment`。
  - 英文部署状态从 `FRONT` / `DEPTH` / `GARRISON` 改为 `Front` / `Depth` / `Garrison`。
  - 唐宋编成比例从百分号改为“占 N／100”。
- `RegionInspectorView`：
  - 英文 `Hex FrontZone` / `FrontZone` 改为 `Hex Front Zone` / `Front Zone`。
  - 英文资源缩写 `MP/IC/SUP` 改为 `Manpower/Industry/Supplies`。
  - 唐宋围城摘要改为攻方、守方、压力、城防和围城军队数量的结构化读法。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 并发子 Agent

- Unit/Region/GeneralProfile 扫描指出检查面板英文内部型标签、唐宋编成百分号、围城摘要和 GeneralProfile 读屏仍可硬化。
- MapEditor/Tooltip 扫描指出 UnitTooltip accessibility value、英文 low supply 文案、MapEditor 画布 value 和底图说明仍可硬化。
- 本轮只纳入 UnitTooltip、UnitInspector 和 RegionInspector，GeneralProfile 与 MapEditor 继续留作后续低风险小切片。

## 禁止项

- 不改变 `Division`、`ComponentType`、`RegionInspectorState`、`SiegeRecord`、围城规则、补给规则、命令、AI 决策或 Codable schema。
- 不改变 SpriteKit 棋盘 custom actions 或逐 hex focus 行为。
- 不做完整 VoiceOver 实机验收、截图验收、横竖屏布局验收或发布级 UI 验收。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成发布级 UI 验收完成。

## 遗留风险

- GeneralProfile 关闭按钮、档案指标 value、辖下军队读屏、MapEditor 画布 value 和底图说明仍可继续拆成后续小切片。
- 逐地块/逐军队 VoiceOver focus order、完整截图、iPhone/iPad 横竖屏布局、完整 VoiceOver 实机和 v5.9 发布验收仍待后续。
- 结构化 event payload、全项目写入端本地化和真 LLM 输出本地化仍未完成。
