# v5.8ai 战报胜负与评分卡片读屏硬化记录

## 目标

继续收口唐宋默认主路径中的战报面板，让顶部“胜负”和“评分估算”两张复合卡片在读屏时作为完整摘要朗读，减少标题、胜负原因、分数、档位和说明被拆成多个碎片焦点的问题。

## 实现范围

- `EventLogView`：
  - 胜负卡片补充 `.accessibilityElement(children: .ignore)`。
  - 新增 `victorySummaryAccessibilityLabel()` 和 `victorySummaryAccessibilityValue(_:)`，唐宋路径 label 读作“胜负摘要”，value 复用既有 `victorySummary`。
  - 评分估算卡片补充 `.accessibilityElement(children: .ignore)`。
  - 新增 `settlementAccessibilityLabel()` 和 `settlementAccessibilityValue(for:)`，value 复用既有 `SettlementSummary.score`、`grade` 和 `detail`。
  - 胜负显示、评分估算公式、卡片视觉和数据来源保持不变。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 并发子 Agent

- UI 只读扫描确认 v5.8ah 后 `EventLogView` 顶部胜负、评分估算和胜利目标行仍适合继续做逐卡片 accessibility 小切片。
- 本轮主线程选择更窄的胜负卡片与评分估算卡片，不扩大到胜利目标行，降低焦点语义变化范围。
- 文档口径沿用 v5.8ag / v5.8ah：只记录已落地的展示层可读语义，不声明完整 VoiceOver 实机或发布级 UI 验收完成。

## 禁止项

- 不改变 `VictoryState`、`VictoryReason`、`VictoryRules.objectiveProgress(in:)`、`VictoryObjectiveProgress` 或评分估算算法。
- 不改变 `GameLogEntry`、`CommandResultLogEntry`、事件写入端、日志结构或 Codable schema。
- 不改变 `TurnReportSummary` 派生逻辑、`TangSongEventLogMessage` 显示桥语义或结构化 event payload。
- 不改变 `Command`、`CommandValidator`、`CommandExecutor`、`RuleEngine`、`WarCommandExecutor`、AI 决策或 hex/region/theater/front/deploy 权威边界。
- 不扩大到胜利目标行、HUD 卡片、主棋盘 focus tree 或全项目 accessibility 扫描。
- 不声明完整 VoiceOver 实机验收、截图布局验收、全项目玩家可见残留清零或发布级 UI 完成。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成完整全项目写入端本地化、结构化 event payload 或发布级 UI 验收完成。

## 遗留风险

- 胜利目标行仍未做逐行合并读屏审计。
- 完整 VoiceOver 实机、截图布局、全项目玩家可见残留扫描、真 LLM 输出本地化和发布级 UI 验收仍未完成。
