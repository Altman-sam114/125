# v5.8ag 战报行整行读屏硬化记录

## 目标

继续收口唐宋默认主路径中的战报面板，让最近战报列表行在读屏时作为一条完整战报朗读，减少分类徽标、回合元数据和正文被拆散朗读的问题。

## 实现范围

- `EventLogView`：
  - 最近战报列表行补充 `.accessibilityElement(children: .ignore)`。
  - 新增 `logEntryAccessibilityLabel(for:)`，唐宋路径读作“战报：战斗 / 粮道 / 州府”等分类。
  - 新增 `logEntryAccessibilityValue(for:)`，复用 `metadata(for:)` 与 `displayMessage(for:)` 拼接回合、势力、阶段和正文。
  - 唐宋正文仍经 `TangSongEventLogMessage.display` 显示桥，不新增日志解析或写入。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
  - 将 v5.8af 的云端通过结果补写到 `update_log.md`。

## 并发子 Agent

- 主游戏 UI 只读扫描确认 `EventLogView` 最近战报行由分类、metadata 和正文多个 `Text` 构成，行容器此前没有合并 accessibility 语义。
- 文档扫描确认 v5.8af 阶段记录和遗留风险已把 `EventLogView` 战报行整行读屏列为后续候选，并建议本轮新增 `v5.8ag_event_log_row_accessibility_record.md`。

## 禁止项

- 不改变 `GameLogEntry`、`CommandResultLogEntry`、事件写入端、日志结构或 Codable schema。
- 不改变 `TangSongEventLogMessage` 既有显示桥语义，不新增结构化 event payload。
- 不改变 `Command`、`CommandValidator`、`CommandExecutor`、`RuleEngine`、`WarCommandExecutor`、AI 决策或 hex/region/theater/front/deploy 权威边界。
- 不扩大到战报摘要卡片、胜负卡片、目标卡片、主棋盘 focus tree 或全项目 accessibility 扫描。
- 不声明完整 VoiceOver 实机验收、截图布局验收、全项目玩家可见残留清零或发布级 UI 完成。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成完整全项目写入端本地化、结构化 event payload 或发布级 UI 验收完成。

## 遗留风险

- 本回合摘要、胜负摘要、胜利目标卡片等复合区域仍未做逐卡片合并读屏审计。
- 完整 VoiceOver 实机、截图布局、全项目玩家可见残留扫描、真 LLM 输出本地化和发布级 UI 验收仍未完成。
