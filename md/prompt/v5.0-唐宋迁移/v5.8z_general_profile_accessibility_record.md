# v5.8z 将领档案 accessibility 与 fallback 硬化记录

## 目标

继续收口 `GeneralProfileView` 的读屏语义和唐宋 fallback，让将领档案的关闭、指标、技能、辖下军队、HQ 警告和缺名状态更适合 VoiceOver 阅读，并减少默认唐宋路径的 Latin 残留。

## 实现范围

- `GeneralProfileView`：
  - 关闭按钮补充“关闭将领档案 / Close General Profile” accessibility label 和返回提示。
  - 忠诚与军心 `metricBar` 合并为单个可访问元素，value 显示“满百 / out of 100”。
  - 技能项补“特长 / Skill” accessibility label。
  - HQ 受压状态补“警告”读法。
  - 辖下军队行补完整 accessibility label/value，唐宋路径读作军队名与“兵力 x，满额 y”。
  - 唐宋头像占位遇到 legacy Latin 名称时显示“将”，避免继续显示 Latin initials。
  - 唐宋所辖方面名为空或含 Latin 时 fallback 为“未命名方面”。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 并发子 Agent

- GeneralProfile 只读扫描指出关闭按钮缺上下文、指标进度条缺显式 value、辖下军队比例缺上下文、HQ 警告依赖颜色/图标、技能项可补完整读法，头像 initials 与方面名 fallback 仍可继续硬化。
- 文档位置由主线程按 v5.8x/v5.8y 相邻记录格式同步；未采用无可用正文的子 Agent 文档输出。
- 本轮只纳入 GeneralProfile 单文件 UI 语义，不扩展到 MapEditor 画布 value、底图按钮说明、响应式布局或完整 Dynamic Type 调整。

## 禁止项

- 不改变 `GeneralData`、`GeneralAssignment`、`FrontZone`、`Division`、将领分配、玩家计划、命令、规则、AI 决策或 Codable schema。
- 不改变 `RootGameView` 面板切换、SpriteKit 棋盘、逐 hex focus、地图渲染或命中测试。
- 不做完整 VoiceOver 实机验收、截图验收、横竖屏布局验收、Dynamic Type 视觉验收或发布级 UI 验收。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成发布级 UI 验收完成。

## 遗留风险

- GeneralProfile 的响应式横/竖排、身份区合并语义、阵营 badge value、技能长文换行和 Dynamic Type 视觉仍未实机验收。
- MapEditor 画布 value、底图按钮/参数说明、逐地块/逐军队 VoiceOver focus order、完整截图、iPhone/iPad 横竖屏布局、完整 VoiceOver 实机和 v5.9 发布验收仍待后续。
- 结构化 event payload、全项目写入端本地化和真 LLM 输出本地化仍未完成。
