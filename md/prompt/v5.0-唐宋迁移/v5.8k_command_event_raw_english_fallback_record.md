# v5.8k 命令面板与战报 raw 英文兜底硬化记录

完成日期：2026-07-06

## 目标

- 继续收口唐宋默认玩家主路径中的英文/内部 key 外露。
- 优先处理命令面板与战报面板的兜底显示，覆盖旧日志、旧诊断和未归档事件。
- 保持本轮为 UI 显示桥小切片，不改变规则、事件写入职责、AI 决策或 Codable schema。

## 并发子 Agent 发现

- UI/SpriteKit 扫描指出：`CommandPanelView` 兜底会直出原始英文消息，`EventLogView` 未识别战报会原样显示，`RootGameView.nextActionHint` 仍出现 `AI`。
- Rules/App/Turn 扫描指出：退却、被围损耗、动态方面、州府归属和玩家方面军令诊断仍可能以英文写入日志或诊断。
- 本轮只选择命令/战报兜底显示作为切片，未处理全部写入端和 AI 面板调试区残留。

## 实现内容

- `TangSongEventLogMessage` 改为模块内可复用，供命令面板调用。
- `CommandPanelView.commandMessageText` 的唐宋最终兜底改为 `TangSongEventLogMessage.display(message)`。
- `EventLogView` 补中文映射：
  - `retreated from ... to ...`
  - `suffered encirclement attrition`
  - `AI turn requested`
  - `Player directive generated no executable commands`
  - `command(s) were rejected by rules`
  - `micromanaged division(s) excluded`
  - `Region ... controller changed to ...`
  - `Hex ... reassigned to dynamic theater ...`
- `EventLogView` 若唐宋显示桥处理后仍含拉丁字母，显示中文兜底：“战报已更新；原始记录留在调试日志中。”
- `RootGameView.nextActionHint` 已行动提示改为“结束回合让各方军议推进”。

## 非目标

- 不改 `GameLogEntry.message` 的存储内容。
- 不改 `SupplyRules`、`CommandExecutor`、`StrategicStateSynchronizer`、`TurnManager` 等写入端。
- 不改 `Command`、`CommandValidator`、`RuleEngine`、`WarCommandExecutor`、AI 决策或任何 hex / region / theater / front / deploy 权威边界。
- 不新增结构化 event payload，不改 JSON/Codable schema。

## 验证口径

- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 提交并推送 `origin/main` 后，由 GitHub Actions `WWIIHexV0 CI Results` 云端验证。
- Agent C 核对本轮最新 commit 对应 artifact 的 manifest、JUnit、主构建日志、failure summary 和 static checks。

## 遗留风险

- 写入端仍可能产生英文日志；本轮只是显示层兜底，后续应逐步源头唐宋化或引入结构化事件。
- AI 面板 diagnostics/raw JSON、GeneralCommandPanelView 计划 raw id、DiplomacyPanelView 数据名 Latin guard、macOS 菜单与 SpriteKit 空地图英文提示仍未在本轮处理。
