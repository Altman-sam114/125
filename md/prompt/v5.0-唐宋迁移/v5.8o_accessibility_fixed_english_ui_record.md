# v5.8o 固定英文与 ASCII UI 硬化记录

完成日期：2026-07-06

## 目标

- 继续收口唐宋默认玩家路径中的固定英文、ASCII 分隔符、raw id 样式显示和 debug-like 读法。
- 优先处理低耦合 UI 面板中仍可见的 `/`、`, `、`q,r` 等非唐宋读法。
- 保持本轮为将领面板与州府详情面板的只读显示桥小切片，不改变规则、命令或数据结构。

## 并发子 Agent 发现

- `GeneralCommandPanelView` 将领副标题仍用 ASCII ` / ` 连接 rank 与用兵风格。
- `GeneralCommandPanelView` 唐宋“已拟军令”仍显示“进攻 / 目标”或“固守 / 目标”。
- `RegionInspectorView` 唐宋地块坐标仍显示为 `q,r`。
- `RegionInspectorView` 唐宋目标列表和军队列表仍用 `, ` 拼接。
- `RegionInspectorView` 唐宋围城城防仍使用 ASCII `/`。

## 实现内容

- `GeneralCommandPanelView` 新增场景感知副标题 helper，唐宋路径使用“职衔 · 用兵风格”，legacy 路径保留 `rank / style`。
- 唐宋已拟军令摘要改为“进攻：目标”或“固守：目标”，不再使用 ASCII `/`。
- `RegionInspectorView` 唐宋地块坐标改为“第 q 列，第 r 行”。
- 唐宋目标列表、己方军队和可见敌军列表改用“、”连接。
- 唐宋围城城防读法改为 `城防 当前／上限`。

## 非目标

- 不改 `PlayerPlannedOperation`、`RegionInspectorState`、`SiegeRecord`、`Division`、`RegionNode` 或 Codable schema。
- 不改 `Command`、`ZoneDirective`、`WarCommandExecutor`、`CommandValidator`、`RuleEngine`、AI 决策或事件写入职责。
- 不宣称全项目已无英文或 ASCII；本轮只覆盖两个高频唐宋面板的小切片。
- 不做完整 VoiceOver 实机验收、截图验收、iPhone/iPad 横竖屏布局验收或发布级 UI 验收。

## 验证口径

- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 提交并推送 `origin/main` 后，由 GitHub Actions `WWIIHexV0 CI Results` 云端验证。
- Agent C 核对本轮最新 commit 对应 artifact 的 manifest、JUnit、主构建日志、failure summary 和 static checks。

## 遗留风险

- 全局 accessibility label/value/hint、截图/布局/VoiceOver 和发布级 UI 验收仍未完成。
- `AppContainer` 写入端仍保留部分 legacy 英文日志，唐宋路径目前主要由 `EventLogView` / `CommandPanelView` 显示桥兜底；后续可继续把高频写入端中文化。
- SpriteKit 棋子、经济面板、MapEditor 和其他非默认路径仍需后续玩家可见残留扫描。
