# v5.8an 外交面板复合行读屏硬化记录

## 1. 目标

继续推进 `codex-v5.0-唐宋aiagent历史策略迁移总提示词.md` 的 v5.8 发布候选硬化，选择一个低风险、可独立验证的 SwiftUI accessibility 小切片：让 `DiplomacyPanelView` 的诸国、关系和最近归附记录复合行具备完整读屏上下文。

本轮按用户要求继续使用并发子 Agent 加速，只读扫描候选面板；主线程负责最终改动、文档同步、commit、push 和云端 artifact 核对。

## 2. 并发子 Agent 结论

- `DiplomacyPanelView.swift` 子 Agent：建议补诸国列表行、外交关系列表行和最近归附记录行的 row-level accessibility，避免国家副标题、关系状态和归附详情被拆成碎片朗读。
- `AgentPanelView.swift` 子 Agent：建议后续处理“诏令朝议”、方面军令标题、directive card 和 raw JSON 调试块的读屏语义，本轮不展开。
- `RootGameView.swift` / `HUDView.swift` 子 Agent：建议后续补亲征选择器 label、棋盘 custom action 候选 gating 和 HUD 结束回合提示，本轮不展开。

本轮选择 `DiplomacyPanelView`，因为它只涉及一个文件内的 SwiftUI accessibility 修饰符和既有显示桥复用，不触碰外交规则、天命、命令、AI 或 schema。

## 3. 实现范围

关键源码：

- `WWIIHexV0/UI/DiplomacyPanelView.swift`

改动：

- 诸国列表行增加 accessibility label/value：
  - label 复用 `countryDisplayName(_:)`
  - 唐宋 value 读出政权、集团和战意
- 外交关系列表行增加 accessibility label/value：
  - label 复用 `relationTitle(_:)`
  - 唐宋 value 读作“关系状态：盟好 / 称臣 / 协战 / 中立 / 敌对 / 交战 / 归附中 / 议和”
- 最近归附记录行增加 accessibility label/value：
  - label 复用 `pacificationTitle(_:)`
  - 唐宋 value 读出结果状态，并复用 `pacificationDetail(_:)` 的回合、天命变化和归附州府详情
- 新增私有 helper：
  - `countryAccessibilityValue(_:)`
  - `relationAccessibilityValue(_:)`
