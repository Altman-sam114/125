# v5.8i 命令反馈与战报元数据硬化记录

## 目标

- 延续 v5.8 发布候选硬化，减少唐宋默认主路径中命令反馈、AI 回合反馈、方面军令反馈和战报 metadata 的 raw validation / raw id / 英文残留。
- 保持 `CommandValidationError` raw case、`CommandResult` schema、`GameLogEntry` schema 和规则判定不变。
- 把 Unit/Region inspector raw id 另列后续切片，避免本轮 UI 范围扩散。

## 非目标

- 不改 `CommandValidator` 的合法性规则，不改 `RuleEngine` 的执行语义。
- 不改 `WarCommandExecutor`、AI 决策、`ZoneDirective`、`WarDirectiveRecord` 或 `AgentDecisionRecord` schema。
- 不做完整结构化 event payload，不删除 `relatedRecordId` 字段，只在唐宋战报 metadata 隐藏内部 id。
- 不处理 Unit/Region inspector、MapDisplayAdapter objectiveStatus、截图/布局或完整 VoiceOver 验收。
- 不做本机测试、build、Swift parse、Markdown 检查或 `git diff --check`。

## 实现说明

- `CommandValidationError.displayName(isTangSongScenario:)` 为唐宋场景提供中文拒绝原因，`RuleEngine` 只在显示消息中使用该桥。
- `AppContainer` 的 AI 回合反馈改为“军议回合已完成 / 尚有 N 项问题”，方面军令反馈改为“方面军令已执行 N 道命令”，提交日志使用防区显示名。
- `AppContainer.selectionMessage` 唐宋路径写入“已选州府 / 已选地块”，不再把 `RegionId.rawValue` 写进交互日志。
- `CommandPanelView` 对 `lastCommandMessage` 增加唐宋兜底显示桥，处理旧英文 AI/General order 文案和常见 raw validation key。
- `EventLogView.metadata(for:)` 在唐宋场景下隐藏 `relatedRecordId`，legacy 阿登路径仍保留旧 metadata。

## 并发子 Agent 结果

- 命令反馈 explorer 只读确认最小文件集是 `AppContainer.swift`、`CommandPanelView.swift`、`EventLogView.swift`，并指出 `selectionMessage` 写入 region raw id；本轮已处理。
- inspector explorer 只读确认 Unit/Region inspector 的 region/theater/frontZone raw id、frontLineSummary、supplyRouteSummary 与 MapDisplayAdapter objectiveStatus 仍是下一轮高价值残留；本轮记录为 v5.8j 候选。

## 验证口径

- 按用户当前要求，本轮不在本机运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 本轮完成后直接 commit 并 push 到 `origin/main`，触发 GitHub Actions `WWIIHexV0 CI Results`。
- Agent C / 主线程验收必须下载最新 `origin/main` commit 对应的未加密 artifact，核对 `ci-artifact-manifest.json` 的 `branch`、`commitSha`、`runId`、`runAttempt`，并阅读 `junit.xml`、`static-checks.log`、`xcodebuild.log` 和 `ci-failure-summary.md`。

## 遗留风险

- Unit/Region inspector 仍可能在唐宋路径显示 raw region/theater/frontZone id；MapDisplayAdapter objectiveStatus 仍可能生成英文状态。
- 部分低频交互日志 caller 仍保留英文 fallback，后续应按玩家可见频次继续切片收口。
