# v5.8q - AppContainer 源头反馈中文化记录

完成日期：2026-07-06

## 1. 目标

本轮继续 v5.8 发布候选硬化，聚焦 `WWIIHexV0/App/AppContainer.swift` 的玩家交互反馈写入端。目标是在唐宋默认路径下减少英文拒绝语、ASCII 坐标、`enemy/non-hostile` 标签和命令 raw id 摘要进入 `lastCommandMessage` / `interactionLog`，让常见反馈从源头写入唐宋文案，而不是完全依赖 `EventLogView` / `CommandPanelView` 后置显示桥兜底。

## 2. 已完成

- `submit(_:)` 唐宋路径的 `lastCommandMessage` 改为动作级中文反馈：行军、进攻、围城、修城、解围、招降、招抚、固守、准许退却、整补、府库军备和结束回合。
- `submit(_:)` 唐宋交互日志改用中文冒号与动作级摘要，规则拒绝原因继续读取 `CommandValidationError.displayName(isTangSongScenario:)`。
- 固守、准许退却、整补、围城、修城、解围、招降、招抚的本地 guard 拒绝语在唐宋路径改为中文写入。
- 将领军令入口、玩家军令阶段、来源方面归属、刷新后方面变化和府库观战拒绝在唐宋路径改为中文写入。
- 观战查看军队、选中本方军队、选中敌方/非敌对军队和选中地块坐标改为唐宋读法；地块坐标使用“第 q 列，第 r 行”。
- 非唐宋路径保留 legacy 英文 fallback，便于阿登历史回归。
- README、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md` 已同步 v5.8q 状态。

## 3. 非目标

- 不改 `Command`、`CommandResult`、`GameLogEntry`、`CommandValidation` 或 Codable schema。
- 不改 `RuleEngine`、`CommandValidator`、`CommandExecutor`、`WarCommandExecutor` 的规则语义。
- 不引入结构化 event payload。
- 不处理真实 LLM 输出、AI raw JSON、全局 accessibility、VoiceOver 实机、截图或布局验收。
- 不宣称全项目无英文、无 raw id 或无 ASCII；本轮只覆盖 AppContainer 常见玩家交互反馈源头。

## 4. 云端验证口径

- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 完成后直接 commit 并 push 到 `origin/main`。
- 由 GitHub Actions `WWIIHexV0 CI Results` 对 `origin/main` 最新 commit 做云端重验证。
- Agent C / 主线程验收时必须核对未加密 artifact 中的 `ci-artifact-manifest.json`，确认 `branch`、`commitSha`、`runId`、`runAttempt` 与最新 `origin/main` 完全一致，并阅读 `junit.xml`、`static-checks.log`、`xcodebuild.log` 和 `ci-failure-summary.md`。

## 5. 后续风险

- EventLog 写入端仍不是结构化 payload，历史日志和其他来源仍可能写入旧英文或 raw id。
- 真 LLM 输出、AI diagnostics/raw JSON、外部资源授权、截图布局和 VoiceOver 仍需后续 v5.8/v5.9 小切片或发布验收处理。
- 底层 legacy enum / schema 名称仍保留，这是兼容桥，不在本轮迁移。
