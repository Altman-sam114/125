# v5.6d 唐宋天命胜利评价桥记录

本文记录唐宋 v5.6d 小切片：让天命/国威进入 `VictoryRules` 的唐宋专用胜利评价。它不是完整统一结算、治理政策或归附纳土实现。

## 1. 目标

- 唐宋场景不再套用 Bastogne / St. Vith 的 legacy 阿登胜利条件。
- 宋统一胜利同时读取关键州府 objective 控制方和宋天命。
- 割据生存同时读取核心都城 objective 控制方、最大回合和割据天命。
- 保持阿登 legacy 胜利逻辑不变。

## 2. 规则口径

```text
唐宋宋胜利：
  控制 开封 / 洛阳 / 太原 / 金陵 / 成都 / 杭州 中至少 4 处
  且 MandateState.legitimacy(.allies) >= 60
  -> VictoryReason.tangSongUnificationByMandate

唐宋割据生存：
  turn >= maxTurns
  且 .germany 控制 太原 / 金陵 / 成都 中至少 2 处
  且 MandateState.legitimacy(.germany) >= 35
  -> VictoryReason.tangSongSeparatistSurvival
```

该规则仍使用当前 `.allies` = 宋、`.germany` = 割据 AI 的 legacy bridge。它只读取 `MapState.controllerOfObjective(named:)` 和 `MandateState`，不直接改地图、外交或部队。

## 3. 非目标

- 不读取 JSON `victoryConditions` 动态生成规则。
- 不新增胜利等级、评分、统一评价面板或战报详情 UI。
- 不实现治理政策、民心、治安、税粮、叛乱。
- 不交割归附后的 hex / region controller，不转换部队。
- 不同步 `DiplomacyState` 与 `TurnOrderState.relations`。
- 不改变 `MandateState` 的调整来源；当前仍主要由 `Command.proposeSubmission` 推动。

## 6. 遗留风险

- 唐宋胜利条件目前写在 `VictoryRules` 代码中，没有读取 JSON `victoryConditions` 的 `status` / `description`。
- HUD 仍只显示 winner，不显示唐宋 victory reason 的中文说明。
- 军事占领本身不会自动增加天命；天命主要来自归附提议，后续若要让军事统一提高国威，需要单独设计规则层事件。
- 多政权仍通过 `.allies/.germany` 桥接，不能表达吴越等单国的独立胜负状态。
