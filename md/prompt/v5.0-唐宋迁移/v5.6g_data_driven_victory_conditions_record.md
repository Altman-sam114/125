# v5.6g 数据驱动唐宋胜利条件记录

本文记录唐宋 v5.6g 小切片：把 `jianlong_960_unification` 场景 JSON 中已有的 `victoryConditions` 接入运行态和 `VictoryRules`，让唐宋胜利判断优先按 objective id、count、turn 和 mandate threshold 读取数据，而不是继续硬编码中文州府名。

## 1. 目标

- `GameState` 保存从场景 JSON 读取的 `victoryConditions`，旧存档缺字段时兼容为空数组。
- `VictoryConditionDefinition` 支持可选 `mandateThreshold`，唐宋场景用数据表达天命门槛。
- `VictoryRules.updateTangSongVictoryState` 优先读取 `state.victoryConditions`，支持当前唐宋 JSON 的 `controlObjectives` 与 `holdObjectives`。
- objective 控制查询改用 objective id，不再依赖中文名称匹配。
- 默认 `loadGameState(...)` 路径会校验 scenario objective 引用、胜利条件 faction/type/status/count 和 objective id，避免坏 JSON 被静默跳过。
- 阿登 legacy 胜利规则保持原样；唐宋条件缺失时保留 v5.6d 硬编码 fallback。

## 2. 实现口径

```text
tangsong_jianlong_960_scenario.json victoryConditions
  -> ScenarioDefinition.VictoryConditionDefinition
  -> DataLoader.loadGameState
  -> DataLoader.validateScenarioReferences
  -> GameState.victoryConditions
  -> VictoryRules.updateTangSongVictoryState
  -> MapState.controllerOfObjective(id:)
  -> VictoryState.winner / reason
```

当前支持的唐宋条件：

- `controlObjectives`: 指定 faction 控制 `objectiveIds` 中至少 `count` 个；若设置 `mandateThreshold`，还要求该 faction 天命不低于阈值。
- `holdObjectives`: 指定 faction 在 `turn` 或 `turns` 达到后仍控制 `objectiveIds` 中至少 `count` 个；若设置 `mandateThreshold`，还要求天命不低于阈值。

`status` 当前只映射已有原因：

- `majorVictory` -> `.tangSongUnificationByMandate`
- `survival` -> `.tangSongSeparatistSurvival`

## 3. 非目标

- 不泛化阿登 legacy 胜利条件，不改 Bastogne / St. Vith / 单位损失 / 装甲断补逻辑。
- 不新增完整评分、评价等级、胜利面板 UI 或战报详情。
- 不实现治理政策、归附纳土、单国胜负或国家级 `PowerId`。
- 不改变 `MandateState` 调整来源。
- 不让 AI、UI 或事件绕过 `VictoryRules` 直接写胜负。

## 5. 并发审计

本轮按用户要求继续使用并发子 Agent 加速：

- 数据驱动胜利审计子 Agent 确认唐宋 JSON 已有 `controlObjectives` / `holdObjectives`，但运行态未保存条件，`VictoryRules` 仍硬编码中文 objective name、count 和天命阈值。
- 归附/治理审计子 Agent 确认治理政策可作为后续小切片，但真实纳土会触碰 hex/controller/theater/front/deploy，风险高于本轮数据驱动胜利条件切片。

主线程选择 v5.6g 数据驱动胜利条件，避免本轮扩大到控制权交割或治理命令。

## 7. 遗留风险

- `status` 仍是 JSON 字符串，但默认加载校验会拒绝当前不支持的 status；本轮只支持当前唐宋 JSON 使用的 `majorVictory` 与 `survival`。
- 数据驱动条件只覆盖 objective 控制、回合门槛和天命阈值，不覆盖归附纳土、治理评价、评分档位或多国家胜利。
- 旧存档或测试构造若没有 `victoryConditions`，仍走 v5.6d fallback；后续若要完全移除 fallback，需要先迁移测试夹具和旧存档策略。
