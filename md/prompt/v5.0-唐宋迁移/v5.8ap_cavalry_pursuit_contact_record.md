# v5.8ap 唐宋骑军追击与将领档案接触口径记录

## 1. 本轮目标

在 v5.8ao 已把“古代作战不存在现代连续战线、骑兵强大、粮草补给重要、后续参考 CK3 等中古战略游戏继续研究”的方向写入 README / AGENTS 后，本轮继续做一个低风险规则小切片：

- 让唐宋骑军在合适地形下对自动退却敌军造成追击损耗。
- 将将领档案旧英文履历 fallback 从“战线形势”改成“敌我接触形势”。
- 不删除内部 legacy `FrontLine` / `FrontZone` schema，不新增大型 CK3 式战争系统。

## 2. 已完成实现

- `WWIIHexV0/Rules/CommandExecutor.swift`
  - `resolveCombatResult` 新增 `attacker` 参数，攻击与反击结算都传入当前攻击方。
  - 新增 `tangSongPursuitDamage(attacker:defender:baseDamage:in:)`。
  - 唐宋场景下，攻击方是骑军、目标自动退却且目标所在环境为平原，或攻守相邻地格存在道路的丘陵时，追加追击兵力损耗。
  - 城池、关隘、森林和山地不触发该追击。
  - 被围撤退追加损耗与骑军追击损耗累加，不再互相覆盖。

- `WWIIHexV0/UI/GeneralProfileView.swift`
  - 唐宋旧英文履历 fallback 改为“按州府、粮道与敌我接触形势调度军队”。

## 4. 验收边界

本轮严格遵守用户要求：不在本地测试，不运行本地 build、Swift parse、Markdown 检查、`git diff --check`、模拟器、截图或 VoiceOver 实机验收。

预期验收只看 GitHub Actions 云端结果包：

- workflow：`WWIIHexV0 CI Results`
- branch：`main`
- manifest 必须匹配最新 `origin/main` commit
- 需核对 `ci-artifact-manifest.json`、`junit.xml`、静态检查日志、`ci-failure-summary.md` 和 `xcodebuild.log`

## 5. 非目标与风险

- 不实现完整 CK3 式战争系统。
- 不新增骑军 AI、粮队、漕运、仓储容量、会战窗口或自动破城。
- 不重命名 `FrontLine` / `FrontZone` / `hexToFrontZone` 等内部 legacy schema。
- 追击 35% 系数与触发条件只是首轮可构建规则，需要后续结合云端构建、实际战局手感和 AI 表现继续调参。
