# v5.1 多势力与通用回合地基记录

本文记录 v5.1 第一轮实现：只建立多势力、关系和通用回合的兼容地基，不切换默认唐宋剧本，不删除阿登 legacy 数据，不重命名全部业务类型。

## 1. 目标

- 新增 `PowerId`、`PowerProfile`、`PowerRelation`、`TurnOrderState`、`WarRelationRules` 兼容合同。
- 让旧 `Faction.germany/allies` 可以桥接到 `PowerId`。
- 让 `CommandValidator.phaseAllowsCommands` 不再直接 switch `.germanAI -> .germany` / `.alliedPlayer -> .allies`。
- 让 `CommandExecutor.executeEndTurn` 不再直接写死 `germany -> allies -> germany`。
- 让 `AppContainer` 和 `TurnManager` 的 AI/玩家控制判断读取 `TurnOrderState`。

## 2. 非目标

- 不切默认数据到 `jianlong_960_unification`。
- 不新增唐宋 JSON。
- 不删除 `Faction.opponent`，但本轮不再给主命令校验新增依赖。
- 不把所有 `Division`、`ProductionKind`、`EconomyResources` 类型改名。
- 不实现围城、粮道、天命、归附。
- 不跑本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。

## 3. 关键改动

### 3.1 兼容合同

关键文件：`WWIIHexV0/Core/Faction.swift`

新增：

- `PowerId`：ASCII power id，当前 legacy 为 `germany` / `allies`，后续唐宋可用 `power_song` 等 id。
- `PowerControlMode`：`human` / `ai` / `inactive`。
- `PowerProfile`：势力显示、控制模式和 legacy `Faction` 桥。
- `PowerRelationStatus` / `PowerRelation`：多势力关系，含 `allied`、`tributary`、`neutral`、`hostile`、`atWar`、`submitting`、`negotiating`。
- `TurnOrderState`：power order、active power、round、phase、profiles、玩家控制 power、关系表。
- `WarRelationRules`：根据 `PowerRelation` 判断能否攻击；没有关系记录时 fallback 为旧逻辑“非同 faction 可攻击”。

### 3.2 GameState 接入

关键文件：`WWIIHexV0/Core/GameState.swift`

- `GameState` 新增 `turnOrderState`，解码旧存档/旧测试状态时 fallback 到 `TurnOrderState.legacy(...)`。
- 新增 `effectiveTurnOrderState`，当旧测试或旧代码直接修改 `activeFaction` / `phase` 时，规则层能按当前字段临时归一化，避免状态不同步导致误拒绝。

### 3.3 回合校验与推进

关键文件：

- `WWIIHexV0/Rules/CommandValidator.swift`
- `WWIIHexV0/Rules/CommandExecutor.swift`
- `WWIIHexV0/Turn/TurnManager.swift`
- `WWIIHexV0/App/AppContainer.swift`

变化：

- `CommandValidator.phaseAllowsCommands` 改为读取 `state.effectiveTurnOrderState.allowsCommands(...)`。
- 攻击阵营校验改为 `WarRelationRules.canTarget(...)`，保留 legacy fallback。
- `CommandExecutor.executeEndTurn` 改为 `TurnOrderState.advancedAfterEndTurn(...)` 推进 active power / phase / round，再桥接回 legacy `activeFaction`。
- `TurnManager.isAITurn` 改为读取 `TurnOrderState`，不再 switch Germany/Allies。
- `AppContainer.shouldRunAI` / `shouldRunAIInSnapshot` 改为读取 `TurnOrderState.shouldRunAI(...)`；observer mode 仍可让玩家控制 power 交给 AI 自动跑。
- `CommandPanelView` / `EconomyPanelView` 的按钮启用条件改为读取通用命令允许状态，不再直接依赖 `.alliedPlayer`。

### 3.4 数据加载

关键文件：`WWIIHexV0/Data/DataLoader.swift`

- `loadGameState(scenarioName:regionName:)` 读取 `playerFaction` / `aiFaction` 后构造 `TurnOrderState.legacy(...)`。
- 默认阿登数据仍保留；v5.2 才切唐宋剧本数据。

## 4. 兼容边界

- `Faction.germany/allies` 仍存在，旧数据、旧测试和旧 UI 仍可运行。
- `GamePhase.germanAI/alliedPlayer` 仍存在，短期只是作为通用 command slot 的 legacy raw value。
- `Faction.opponent` 仍存在，其他规则文件仍有历史引用；后续 v5.1 续作要逐步迁到 `WarRelationRules`。
- `GameState.activeFaction` / `phase` 仍保留给旧管线和 UI；新的权威规划状态是 `turnOrderState`。

## 5. 遗留风险

- 仍未完成全项目敌我判断迁移；`SupplyRules`、`RegionSupplyRules`、`FrontLineManager`、`RegionCombatRules`、`AgentContexts`、`WarCommandExecutor`、`ZoneCommanderAgent` 等仍有 `Faction.opponent` 残留。
- 多政权超过两个 power 时，底层战术对象仍需要 legacy `Faction` 桥；真正多政权落地仍需后续拆 `Faction` 数据驱动。
- 默认启动仍是阿登，不是唐宋首发剧本。
- 玩家可见 UI 仍有 Ardennes、Manpower、Industry、Supplies、Panzer 等残留；这是 v5.2-v5.5 范围。
