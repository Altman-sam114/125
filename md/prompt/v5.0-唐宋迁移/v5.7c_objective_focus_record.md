# v5.7c 目标锚点定位记录

本文记录唐宋 v5.7c 小切片：在 v5.7b 已能把主要统一目标显示为“已据/待取”州府锚点的基础上，让这些锚点成为可点击按钮，点击后只聚焦目标所在 hex / region。

## 1. 目标

- HUD 目标锚点显示为横向小按钮，保留“已据 / 待取”读法。
- 点击目标按钮后选中对应 objective 的 hex 和 region。
- 复用现有 BoardScene selected hex / selected region 高亮，以及 Region inspector 的目标州府详情。
- 帮助新玩家从 HUD 的目标名单快速定位地图上的开封、洛阳、太原、金陵、成都、杭州等关键州府。

## 2. 非目标

- 不新增地图高亮层、镜头移动、路线指引、自动缩放或持续追踪。
- 不提交 `Command`，不调用 `RuleEngine`，不移动单位，不改变战斗/围城/外交合法性。
- 不写 `GameState`、`eventLog`、hex controller、region controller、theater/front/deploy 或 diplomacy。
- 不替代玩家点击地图和州府面板；只是复用现有选区链路。
- 不做截图、模拟器、Xcode build/test 或完整布局验收。

## 3. 实现链路

```text
HUDView.objectiveGuideItems
  -> Button(objective.id)
  -> RootGameView passes onFocusObjective
  -> AppContainer.focusObjective(id:)
  -> selectedHex = objective.coord
  -> selectedRegionId = mapDisplayAdapter.regionId(for: objective.coord)
  -> BoardScene / RegionInspectorView 只读显示目标选区
```

`focusObjective(id:)` 只更新 UI 选区并写一条 interaction log，不改变 `GameState`。

## 7. 遗留风险

- 当前只是选区聚焦，不会自动平移/缩放镜头；目标在当前视口外时，玩家仍需手动拖动地图。
- HUD 信息继续增多，目标按钮横向滚动未做小屏截图、Dynamic Type 或 VoiceOver 完整验收。
- 目标按钮只覆盖主要宋统一 `majorVictory` 条件；割据生存目标和更完整剧本包装仍待后续 v5.7 切片。
