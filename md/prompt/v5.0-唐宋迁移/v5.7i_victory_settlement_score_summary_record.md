# v5.7i 胜负后结算预览/评分估算记录

本文记录唐宋 v5.7i 小切片：在 v5.6h / v5.6i 已有胜负原因与胜利目标进度、v5.7e 已有每回合战报摘要后，为战报面板补上胜负后的轻量结算预览/评分估算。

## 1. 目标

- 唐宋场景胜负已定后，战报面板显示“评分估算”摘要。
- 摘要显示 0-100 的只读估算分数和短档位文案。
- 评分输入只来自已有状态：`VictoryState.winner/reason`、`VictoryRules.objectiveProgress(in:)` 传入且匹配胜者自己的只读快照、当前回合、州府进度和天命门槛。
- 让玩家能在战报面板中快速理解统一或割据生存结果，而不是只看到一句胜负原因。

## 2. 非目标

- 不改变 `VictoryRules.updateVictoryState` 的胜负判定。
- 不写 `VictoryState`、`GameState.eventLog`、存档、hex / region / theater / front / deploy 或 diplomacy。
- 不新增权威结算事件。
- 不实现完整胜利面板、正式评分系统、治理评分、单国胜负、外交纳土结算、自动破城或部队转换。
- 不调用 `RuleEngine`、`CommandExecutor` 或提交任何 `Command`。
- 不做截图、布局、VoiceOver、模拟器或本机 Xcode 验收。

## 3. 实现链路

```text
RootGameView
  -> EventLogView(
       victoryState: gameState.victoryState,
       objectiveProgress: VictoryRules().objectiveProgress(in: gameState),
       currentTurn: gameState.turn
     )

EventLogView.victorySummary
  -> 只读显示 VictoryState.winner / reason

EventLogView.settlementSummary
  -> guard 唐宋场景 + victoryState.winner != nil
  -> 读取匹配胜者的 VictoryObjectiveProgress
  -> 不使用非胜者目标进度作为 fallback
  -> 按州府进度、天命进度、回合速度估算 score
  -> 显示 score / grade / detail
  -> 不写任何状态
```

评分估算只是展示层复盘提示；正式胜负仍只由 `VictoryRules.updateVictoryState` 写入 `VictoryState`。
