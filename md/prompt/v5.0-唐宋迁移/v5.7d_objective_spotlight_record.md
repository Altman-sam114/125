# v5.7d 目标州府地图 spotlight 记录

本文记录唐宋 v5.7d 小切片：在 v5.7b 已有统一目标锚点、v5.7c 已能点击目标聚焦州府的基础上，为 SpriteKit 地图增加只读目标州府 spotlight，让玩家不打开文档也能在地图上识别开封、洛阳、太原、金陵、成都、杭州等主要统一目标。

## 1. 目标

- 地图只读标出主要宋统一目标州府。
- 对目标显示“已据 / 待取”短标签，和 HUD 目标锚点口径一致。
- HUD 点击某个目标后，地图上该目标有额外强调圈。
- 继续复用 `VictoryRules.objectiveProgress(in:)` 与 `MapState.objective`，不新增目标数据来源。
- 保持首屏导览轻量化，不做大段说明页。

## 2. 非目标

- 不新增或修改 victory condition。
- 不改变 `VictoryRules` 判定。
- 不提交 `Command`，不调用 `RuleEngine`，不移动单位，不改变战斗/围城/外交合法性。
- 不写 `GameState`、`eventLog`、hex controller、region controller、theater/front/deploy 或 diplomacy。
- 不做自动镜头移动、自动缩放、路线指引或持续目标追踪系统。
- 不做截图、模拟器、Xcode build/test 或本地轻量检查；本轮按用户要求只走云端 GitHub Actions 验证。

## 3. 实现链路

```text
VictoryRules.objectiveProgress(in:)
  -> MapDisplayAdapter.objectiveOverlays()
  -> ObjectiveOverlayState(id/name/coord/regionId/isControlled)
  -> BoardScene.drawObjectiveOverlays
  -> 地图只读绘制已据/待取 objective spotlight

HUDView objective button
  -> AppContainer.focusObjective(id:)
  -> focusedObjectiveId
  -> BoardRenderState
  -> BoardScene 给当前目标额外强调圈
```

`focusedObjectiveId` 是 UI-only 渲染态，不参与 `CommandValidator`、命令候选、移动、攻击、围城、招抚或 AI 决策。

## 7. 遗留风险

- spotlight 只覆盖主要宋统一 `majorVictory` 条件；割据 survival 目标和多剧本目标包装仍待后续 v5.7 切片。
- 当前只是地图视觉标记和点击后强调，不会自动平移或缩放镜头；目标在当前视口外时仍需玩家手动拖动地图。
- 未做小屏、Dynamic Type、VoiceOver、截图或真实设备布局验收。
- HUD 目标列表和地图 spotlight 都仍通过 objective name 回查 objective；若未来出现重名 objective，需要改为从进度快照直接保留 objective id。
