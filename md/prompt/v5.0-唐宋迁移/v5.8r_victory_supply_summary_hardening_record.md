# v5.8r - 胜利、粮道与军议摘要显示硬化记录

完成日期：2026-07-06

## 1. 目标

本轮继续 v5.8 发布候选硬化，聚焦主游戏默认唐宋路径中仍可能玩家可见的 ASCII 分隔符、坐标格式和上游军议 raw 文本直出风险。范围限定在 HUD、战报、军队详情、府库资源摘要和胜利目标摘要，不改 MapEditor、不改规则、不改数据 schema。

## 2. 已完成

- `VictoryObjectiveProgress.summary(isTangSongScenario:)` 的唐宋路径把 `州府 X/Y`、`天命 X/Y` 改为 `州府 X／Y`、`天命 X／Y`。
- `HUDView` 的统一进度和天命进度在唐宋路径改用“／”。
- `EventLogView` 的胜负后评分估算详情改用“州府 X／Y”“天命 X／Y”。
- `EconomyResources.summary(isTangSongScenario:)` 的唐宋路径改用“丁口 X、钱帛 Y、粮草 Z”，府库军备成本行不再继承英文逗号。
- `UnitInspectorView` 粮道近源坐标在唐宋路径改为“第 q 列，第 r 行”。
- `EventLogView` 对旧英文 `Selected hex q,r.` 日志的唐宋显示桥改为“已选地块：第 q 列，第 r 行。”。
- `EventLogView` 本回合军议摘要对 Latin、JSON 痕迹、schema key、raw id 风险做显示层兜底，改为“军议摘要已形成 / 已形成方面军令”。
- README、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md` 已同步 v5.8r 状态。

## 3. 非目标

- 不改变 `VictoryRules` 的胜负判定语义，只改 `VictoryObjectiveProgress.summary` 显示文本。
- 不改变 `EconomyRules`、资源结算、生产成本或补给判定。
- 不改变 `SupplyRouteSummary`、`AgentDecisionRecord`、`WarDirectiveRecord`、`GameLogEntry`、事件写入职责或 Codable raw 值。
- 不处理 MapEditor 的 raw 文件名、`q,r` 坐标、`JSON` 文案和导出错误 raw id 残留；该批风险留给后续 v5.8s。
- 不处理真 LLM 输出本地化、结构化 event payload、VoiceOver 实机、截图或布局验收。

## 4. 云端验证口径

- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 完成后直接 commit 并 push 到 `origin/main`。
- 由 GitHub Actions `WWIIHexV0 CI Results` 对 `origin/main` 最新 commit 做云端重验证。
- 验收时必须下载未加密 artifact 到 `/private/tmp/wwiihexv0-c-review-<run_id>/`，核对 `ci-artifact-manifest.json` 的 `branch`、`commitSha`、`runId`、`runAttempt`，并阅读 `junit.xml`、`static-checks.log`、`xcodebuild.log` 和 `ci-failure-summary.md`。

## 5. 后续风险

- MapEditor 仍有编辑器可见的 raw 文件名、`q,r` 坐标、`JSON` 技术词、导出错误 raw id 和 legacy 色相 key，可作为 v5.8s 小切片处理。
- AI / LLM 上游仍可能在调试区保留 raw JSON；玩家态已做多层兜底，但完整结构化 event payload 尚未落地。
- 全局截图、布局、VoiceOver 和发布级资源授权验收仍待后续 v5.8 / v5.9 收口。
