# v5.8s MapEditor raw UI 硬化记录

完成日期：2026-07-06

## 目标

继续按唐宋 v5.0-v5.9 总提示词推进发布候选硬化。本轮只处理 MapEditor 玩家/编辑器可见的技术词和 raw 文案残留，降低默认工具路径中 `JSON`、`q,r` 坐标、raw id、完整本机路径和底层错误信息直出。

## 范围

- MapEditor 侧栏资源区、底图区、信息区和状态栏。
- MapEditor 导出校验错误、默认州府/方面 fallback、自动州城/粮仓名称。
- MapEditor 默认唐宋资源桥的可见错误文案。
- README、md 大纲、flow 文档、流程图和 update_log 同步。

## 非目标

- 不改 `ScenarioDefinition`、`RegionDataSet`、`MapEditorDocument` Codable schema 或 JSON key。
- 不改底层 `Faction.allies/germany`、`GamePhase.alliedPlayer`、`RegionId`、`TheaterId`、`Division` 或 legacy raw 值。
- 不改主游戏 `DataLoader`、规则系统、AI 决策、胜利/经济/补给或 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 权限边界。
- 不做结构化 event payload、真 LLM 本地化、全项目写入端本地化、VoiceOver、截图/布局或 v5.9 发布验收。

## 实现摘要

- `HexCoord` 增加 `mapEditorDisplayName`，MapEditor 信息面板、状态栏和导出错误显示“第 q 列，第 r 行”，不把 `q,r` 当默认玩家文案。
- MapEditor 资源区按钮从“导出 JSON 到内存”改为“生成资源预览”，默认资源说明改为“建隆元年剧本 / 州府数据”。
- 底图区只显示底图文件名，不在默认侧栏直接展示完整本机路径。
- 覆盖默认资源、生成预览和写出资源的状态文案改为唐宋资源口径，不默认展示 `.json` 文件名。
- 自动州城名改为顺序编号，粮仓 fallback 改为中文地块读法。
- 导出错误不再直出 `RegionId.rawValue`、`terrain.rawValue` 或底层英文编码错误；缺失资源和读取地形错误也改为中文包装。
- 默认州府/方面 draft 名称 fallback 改为“未命名州府 / 未命名方面”。

## 验证口径

按人工明确要求，本机未运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。本轮完成后应直接 push 到 `origin/main`，由 GitHub Actions `WWIIHexV0 CI Results` 做云端重验证，并由 Agent C 下载未加密 artifact 核对 manifest、JUnit、failure summary、static-checks.log 和 xcodebuild.log。

## 风险与后续

- MapEditor 导出的底层 JSON、`allies/germany`、`alliedPlayer`、`region_#`、`theater_#` 等兼容 raw 值仍保留；这是当前 legacy schema 桥，不在 v5.8s 里伪装成已迁移完成。
- MapEditor 测试夹具和非默认调试路径中的历史命名仍作兼容参考。
- 后续 v5.8/v5.9 仍需继续全局玩家可见残留审计、accessibility、截图/布局和发布验收。
