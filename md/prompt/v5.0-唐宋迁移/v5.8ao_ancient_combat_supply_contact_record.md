# v5.8ao 古代作战、粮道与接触带口径记录

日期：2026-07-07

## 目标

按人工最新要求，把唐宋主线的作战方向从现代连续战线口径继续拉回古代/中古战争：

- 古代作战不存在现代意义的连续战线。
- 骑兵应在开阔地、道路、追击和机动作战中更强。
- 补给、粮草、粮道、仓城、道路、山林、跨河、围城和被围状态必须成为核心规则变量。
- 后续作战模式需要继续研究，可参考《十字军之王3》等中古战略游戏的集结、行军、围城、补给与 attrition 节奏。

## 本轮实际完成

- `AGENTS.md` 增加长期入口规则：唐宋主线不得直接套现代连续战线；骑兵、粮草、粮道和 CK3 类中古战争模型作为后续研究方向；所有落地仍必须适配本项目 hex / region / command / rule 架构。
- `README.md` 增加“唐宋作战模型方向”，并记录 v5.8ao 已落地范围。
- `CombatRules` 把唐宋骑军在开阔地或道路条件下进攻的冲击加成从 `+0.15` 提高到 `+0.30`，继续保留攻城、攻关、攻林地和攻山地的 `-0.15` 抑制。
- `CommandValidator.validateAttack` 与 `validateBesiege` 在唐宋场景下阻止 `SupplyState.encircled` 的军队主动攻击或围城，返回 `CommandValidationError.supplyBlocked`。
- `CommandValidationError.supplyBlocked`、`CommandPanelView` 和 `EventLogView` 补“粮道断绝，不可主动出击”显示桥。
- `MapDisplayLayer` 唐宋图层显示把 `frontLine` 改为“接触带”，把 `deployment` 改为“行营部署”。
- `UnitInspectorView`、`RegionInspectorView` 与 `GeneralCommandPanelView` 把唐宋玩家态“防区 / 部署 / 战线 / 前线压力 / 固守防线”等文案收束为“行营辖区 / 军位 / 敌我接触 / 接触州府 / 接触压力 / 固守城关”等。
- `md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/flow/01_overall_core_flow.mermaid` 与 `update_log.md` 同步 v5.8ao 状态。

## 非目标

- 不删除内部 legacy `FrontLine` / `FrontZone` / `hexToFrontZone` schema。
- 不重构动态战区、部署层、AI 指令 raw case、Codable schema 或存档结构。
- 不实现完整 CK3 式战争系统、真实 rally 点、封建征召、逐粮队运输、仓储容量、自动破城或完整骑军追击模型。
- 不跑本机测试、build、Swift parse、Markdown 检查、模拟器、截图或 VoiceOver 实机验收；按用户要求只走云端 GitHub Actions 验证。

## 架构边界

- Hex 仍是战术权威：单位位置、攻击、移动、占领、补给落点以 hex 为准。
- Region 仍是州府战略聚合层。
- `FrontLine` / `FrontZone` 仍是内部动态接触与部署派生层；唐宋玩家态和后续设计应把它们理解为接触带、敌我接触和行营部署，而不是现代连续战线。
- 玩家、AI 和后续聊天命令仍必须经 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine`，不得绕过规则系统直接改 `GameState`。
