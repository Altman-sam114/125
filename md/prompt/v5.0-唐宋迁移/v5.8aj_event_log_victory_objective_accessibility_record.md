# v5.8aj 战报胜利目标行读屏硬化记录

## 目标

继续收口唐宋默认主路径中的战报面板，让“胜利目标”区域的每条目标进度行在读屏时作为完整目标摘要朗读，减少目标标题、达成状态、进度摘要和详情被拆成多个碎片焦点的问题。

## 实现范围

- `EventLogView`：
  - 胜利目标 progress 行补充 `.accessibilityElement(children: .ignore)`。
  - 新增 `objectiveAccessibilityLabel(for:)`，唐宋路径读作“胜利目标：目标标题”。
  - 新增 `objectiveAccessibilityValue(for:)`，复用既有 `completedText` / `pendingText`、`VictoryObjectiveProgress.summary(isTangSongScenario:)` 和 `detail(isTangSongScenario:)`。
  - 胜利目标可见文本、排序、`progressItems.prefix(3)` 和数据来源保持不变。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/flow/01_overall_core_flow.mermaid` 和 `update_log.md`。

## 并发子 Agent

- UI 只读扫描确认 v5.8ai 后胜利目标 progress 行是下一个最窄且明确的 accessibility 小切片。
- 文档扫描指出 README、md 大纲、flow、flowchart、update_log 和阶段记录需要同步；可选 mermaid 源图也应更新以避免图源口径分叉。
- 本轮主线程只处理 `EventLogView` 的胜利目标行，不扩大到 HUD 目标/下一步、主棋盘 focus tree 或全项目 accessibility 扫描。

## 禁止项

- 不改变 `VictoryState`、`VictoryReason`、`VictoryRules.objectiveProgress(in:)`、`VictoryObjectiveProgress` 计算逻辑、胜利目标排序或 `progressItems.prefix(3)`。
- 不改变评分估算算法、`GameLogEntry`、`CommandResultLogEntry`、事件写入端、日志结构或 Codable schema。
- 不改变 `TurnReportSummary` 派生逻辑、`TangSongEventLogMessage` 显示桥语义或结构化 event payload。
- 不改变 `Command`、`CommandValidator`、`CommandExecutor`、`RuleEngine`、`WarCommandExecutor`、AI 决策或 hex/region/theater/front/deploy 权威边界。
- 不声明完整 VoiceOver 实机验收、截图布局验收、全项目玩家可见残留清零或发布级 UI 完成。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成完整全项目写入端本地化、结构化 event payload 或发布级 UI 验收完成。

## 遗留风险

- HUD 目标/下一步、主棋盘逐地块/逐军队 focus tree 仍未完成完整读屏验收。
- 完整 VoiceOver 实机、截图布局、全项目玩家可见残留扫描、真 LLM 输出本地化和发布级 UI 验收仍未完成。
