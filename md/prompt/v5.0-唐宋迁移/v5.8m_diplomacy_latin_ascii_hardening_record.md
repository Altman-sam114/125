# v5.8m 外交 Latin 名称与 ASCII 连接符硬化记录

完成日期：2026-07-06

## 目标

- 继续收口唐宋默认外交面板中的玩家可见英文、Latin 名称和 ASCII 连接符。
- 覆盖旧存档或 fallback 外交数据带来的 `German Reich`、`United States`、`Axis`、`Allied Coalition` 等 legacy 名称外露风险。
- 保持本轮为外交面板只读显示桥小切片，不改变外交数据结构、命令或规则。

## 并发子 Agent 发现

- `Text(country.name)` 直接显示国家名。
- `LabeledContent(bloc.name)` 直接显示集团名。
- 关系行使用 ` - `。
- 归附记录使用 ` -> `。
- 国家副标题使用 ASCII ` / `。
- 归附详情和君主朝议目标列表使用 ASCII `, `。

## 实现内容

- 新增 `countryDisplayName(_:)`，唐宋路径先按 `CountryId` 映射首发政权与 legacy fallback 国家；其余含 Latin 或等于 raw id 的名称按 faction 兜底为“宋 / 割据政权”。
- 新增 `blocDisplayName(...)`，唐宋路径先按 `DiplomaticBlocId` 映射宋朝廷、抗宋同盟、南方割据诸国和 legacy 集团；其余含 Latin 或 raw id 的集团按 faction 兜底。
- `countryName(_:)` 统一走国家显示 helper。
- 关系行改为“甲 与 乙”。
- 归附记录改为“甲 招抚 乙”。
- 国家副标题分隔符改为“·”。
- 归附州府列表和君主朝议目标列表改为使用“、”。

## 非目标

- 不改 `DiplomacyState`、`CountryProfile`、`DiplomaticBloc`、`PacificationRecord` 或 Codable schema。
- 不改 `Command.proposeSubmission`、`CommandValidator`、`CommandExecutor`、`RuleEngine` 或外交关系投影。
- 不处理 `AgentPanelView` raw JSON、diagnostics、错误原文或 AI summary 的原文风险。

## 验证口径

- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 提交并推送 `origin/main` 后，由 GitHub Actions `WWIIHexV0 CI Results` 云端验证。
- Agent C 核对本轮最新 commit 对应 artifact 的 manifest、JUnit、主构建日志、failure summary 和 static checks。

## 遗留风险

- 当前映射只覆盖首发唐宋政权和已知 legacy fallback id；未来新增国家/集团时仍应补数据驱动 displayName 或更完整的本地化表。
- AI 面板 raw 文本仍需后续 v5.8/v5.9 小切片继续硬化。
