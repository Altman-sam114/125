# v5.8ad 府库军备队列可读性硬化记录

## 目标

继续收口唐宋默认主路径中的府库面板可读性和读屏语义，减少军备队列裸数字、短标签和按钮动作语义不清的问题。

## 实现范围

- `EconomyPanelView`：
  - 唐宋路径的军备队列未就绪项从裸剩余回合数字改为“尚需 N 回合”。
  - 队列就绪项继续显示“就绪”。
  - 府库指标从“入丁 / 入帛 / 耗粮”改为“本回合丁口 / 本回合钱帛 / 本回合耗粮”。
  - 军备按钮补充“下达军备令：军备名” accessibility label。
  - 队列行补充“军备队列：军备名” accessibility label 和剩余回合 accessibility value。
  - 指标行补充合并后的 accessibility label/value。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 并发子 Agent

- 主游戏 UI 只读扫描建议优先选择 `EconomyPanelView`，因为它直接影响玩家主路径府库排产，而 MapEditor 剩余文案更偏低频维护工具。
- 文档只读扫描用于确认 v5.8 路线、flow 当前落地块、流程图时间线和阶段记录清单都需要从 v5.8ac 推进到 v5.8ad。

## 禁止项

- 不改变 `ProductionOrder`、`ProductionKind`、`EconomyRules`、`Command.queueProduction`、资源扣除、队列执行、生产校验、日志结构或 Codable schema。
- 不改变 `onQueueProduction` 调用路径，不新增经济规则，不修改旧存档兼容数据。
- 不声明完整 VoiceOver 实机验收、截图布局验收或发布级 UI 完成。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成完整全项目写入端本地化或发布级 UI 验收完成。

## 遗留风险

- HUD 队列数量、战报摘要计数、将领军令指标和其它低频面板可能仍有短数字、raw id、英文 fallback 或读屏上下文不足，需要继续按 v5.8/v5.9 小切片扫描。
- 完整 VoiceOver 实机、截图布局、全项目玩家可见残留扫描、真 LLM 输出本地化和发布级 UI 验收仍未完成。
