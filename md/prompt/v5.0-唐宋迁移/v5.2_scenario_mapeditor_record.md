# v5.2 唐宋首发剧本数据与默认入口首轮记录

本文记录 v5.2 第一轮实现：让默认主游戏优先加载唐宋 960 剧本数据，并保留阿登 legacy fallback。本轮不是 v5.2 完整收口；MapEditor 术语迁移、完整多政权 `Faction` 数据驱动和发布级视觉仍留后续版本。

## 1. 目标

- 新增 `jianlong_960_unification` 唐宋首发剧本 JSON。
- 新增唐宋州府/方面 region JSON，覆盖默认地图全部 hex。
- 新增唐宋单位模板和人物资料 JSON。
- 让 `DataLoader.loadInitialGameState()` 默认优先读取唐宋资源；阿登仍作 fallback。
- 让默认 HUD / 经济面板 / 命令面板读取唐宋显示桥，减少第一屏二战文案。

## 2. 非目标

- 不删除阿登 JSON、旧测试或 legacy Agent D。
- 不把 `Faction.germany/allies` 一次性扩成完整多政权 enum。
- 不完成 MapEditor 全 UI 术语迁移。
- 不实现围城、粮道、天命、归附或古代军制真实规则。
- 不跑本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。

## 3. 数据产物

新增文件：

- `WWIIHexV0/Data/tangsong_jianlong_960_scenario.json`
- `WWIIHexV0/Data/tangsong_jianlong_960_regions.json`
- `WWIIHexV0/Data/tangsong_unit_templates.json`
- `WWIIHexV0/Data/tangsong_characters.json`

当前规模：

- 14 x 10，共 140 个 hex。
- 70 个 region，全部有 `displayHexes`、`representativeHex`、双向 neighbor 和 theater seed。
- 10 个 theater/front-zone 种子。
- 12 支初始军队。
- 10 个关键 objective / key location。

兼容说明：

- 当前底层 `Faction` 仍只有 `.allies` / `.germany`。唐宋默认数据暂以 `.allies` 表示宋，以 `.germany` 表示北汉、辽边境压力和南方割据 AI 的 legacy 桥。
- `TurnOrderState` 的 `PowerProfile` 将 `.allies` 显示为“宋”，将 `.germany` 显示为“北方与割据诸政权”。
- `tangsong_unit_templates.json` 的古代兵种仍映射到现有 `ComponentType`，真实禁军/厢军/骑军/器械规则留给 v5.3。

## 4. 代码改动

关键文件：

- `WWIIHexV0/Data/DataLoader.swift`
- `WWIIHexV0/Core/GameState.swift`
- `WWIIHexV0/UI/HUDView.swift`
- `WWIIHexV0/UI/EconomyPanelView.swift`
- `WWIIHexV0/UI/CommandPanelView.swift`
- `WWIIHexV0/UI/RootGameView.swift`
- `WWIIHexV0/SpriteKit/BoardScene.swift`
- `WWIIHexV0.xcodeproj/project.pbxproj`

主要变化：

- `loadInitialGameState()` 优先调用唐宋资源：
  - `tangsong_jianlong_960_scenario`
  - `tangsong_jianlong_960_regions`
  - `tangsong_unit_templates`
- `loadGameState(...)` 增加 `unitTemplateName` 参数，旧调用默认仍读 `unit_templates`。
- `makeDivisions(...)` 改为使用传入模板，并读取模板 `maxHP`。
- 唐宋剧本创建定制 `TurnOrderState` 和 `DiplomacyState`，避免默认外交面板仍显示二战国家。
- `GameState` 增加 `scenarioDisplayName`、`phaseDisplayName`、`displayName(for:)` 和 `isTangSongScenario` 显示桥。
- HUD、经济面板、命令面板和棋盘 accessibility label 改为场景感知显示；唐宋路径下资源显示为丁口、钱帛、粮草，生产显示为募厢军、募禁军、募骑军、造器械、整备粮草。
- 新 JSON 加入 iOS `WWIIHexV0` 和 macOS `WWIIHexV0Mac` resources。

## 5. 遗留风险

- MapEditor UI 文案仍未全面迁移为地块、州府、方面、军队/人物。
- 底层 `Faction` 仍是二元 legacy 桥，多政权真实控制权尚未数据驱动。
- AI rationale、事件日志、部分旧面板和规则日志仍可能显示 `Germany` / `Allies` 或二战术语。
- 胜利规则仍是 legacy 规则系统，唐宋 victoryConditions 数据尚未接入专门 `VictoryRules`。
- 本轮未做运行时 UI、模拟器或 XCTest 验证；云端只做静态检查与 Xcode build。
