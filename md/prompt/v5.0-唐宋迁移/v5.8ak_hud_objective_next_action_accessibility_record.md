# v5.8ak HUD 目标与下一步提示读屏硬化记录

## 目标

继续收口唐宋默认主路径的 HUD 可读语义，让首屏“目标”摘要和“下一步”提示在读屏时以完整提示朗读，减少标题与正文被拆成碎片焦点的问题。

## 实现范围

- `HUDView`：
  - “目标”区域的非按钮摘要部分补充 `.accessibilityElement(children: .ignore)`。
  - 新增 `objectiveGuideAccessibilityLabel()`，唐宋路径读作“目标提示”。
  - 新增 `objectiveGuideAccessibilityValue(_:)`，复用既有 `objectiveGuideText`。
  - 目标州府 chips / 按钮继续保留独立 accessibility label/value/hint 和独立焦点，不合并整张目标卡。
  - “下一步”提示卡片补充 `.accessibilityElement(children: .ignore)`。
  - 新增 `nextActionAccessibilityLabel()`，唐宋路径读作“下一步提示”。
  - 新增 `nextActionAccessibilityValue(_:)`，复用既有 `nextActionHint`。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/flow/01_overall_core_flow.mermaid` 和 `update_log.md`。

## 并发子 Agent

- HUD 只读扫描确认 v5.8ae 已覆盖库存/队列指标，v5.8ak 最小切片应优先处理 `HUDView` 的目标摘要和下一步提示卡片。
- 面板只读扫描给出 `AgentPanelView`、`DiplomacyPanelView` 和 `CommandPanelView` 后续候选，但这些候选不并入本轮，避免扩大改动范围。
- 文档只读扫描确认最新口径是 v5.8aj，v5.8ak 需要同步 README、md 大纲、flow、flowchart、mermaid 源图、update_log 和阶段记录。

## 禁止项

- 不改变 `VictoryRules.objectiveProgress(in:)`、`VictoryObjectiveProgress`、目标排序、`objectiveGuideItems` 生成或胜利目标可见文案。
- 不改变 `onFocusObjective`、`AppContainer.focusObjective`、`selectedHex` / `selectedRegionId`、地图 spotlight 或目标州府按钮行为。
- 不改变 `RootGameView.nextActionHint`、`AppContainer.selectedValidatedCommandHint`、移动/攻击高亮、有限合法性预校验或命令候选计算。
- 不改变 `Command`、`CommandValidator`、`CommandExecutor`、`WarCommandExecutor`、`RuleEngine`、AI 决策、日志结构或 Codable schema。
- 不声明完整 VoiceOver 实机验收、截图布局验收、主棋盘逐地块/逐军队 focus tree 完成、全项目玩家可见残留清零或发布级 UI 完成。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成完整 accessibility 验收、结构化 event payload 或发布级 UI 验收完成。

## 遗留风险

- 主棋盘逐地块/逐军队 focus tree、AgentPanel / DiplomacyPanel / CommandPanel 复合状态行读屏仍待后续小切片。
- 完整 VoiceOver 实机、截图布局、全项目玩家可见残留扫描、真 LLM 输出本地化和发布级 UI 验收仍未完成。
