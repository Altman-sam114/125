# v5.8al AI 面板军令结果行读屏硬化记录

## 目标

继续收口唐宋默认主路径的 AI 面板可读语义，让“军令结果”列表中的每条执行结果在读屏时作为完整状态朗读，减少命令标题和执行状态被拆成碎片焦点的问题。

## 实现范围

- `AgentPanelView`：
  - `record.commandResults` 的每条结果行补充 `.accessibilityElement(children: .ignore)`。
  - 新增 `commandResultAccessibilityLabel(_:)`，唐宋路径读作“军令结果：命令标题”。
  - 新增 `commandResultAccessibilityValue(_:)`，复用既有 `resultLine(_:)`。
  - 可见文本、排序、命令结果来源和玩家态/开发态分层保持不变。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/flow/01_overall_core_flow.mermaid` 和 `update_log.md`。

## 并发子 Agent

- AI 面板只读扫描确认 `AgentPanelView` 的 `commandResults` 行是 v5.8al 最小且明确的 accessibility 小切片。
- 命令面板只读扫描给出 `CommandPanelView` 状态行和最近军令反馈候选，但不并入本轮。
- 外交面板只读扫描给出归附记录、诸国行和关系行候选，但不并入本轮。

## 禁止项

- 不改变 `AgentDecisionRecord`、`CommandResultSummary`、`WarDirectiveRecord`、`TheaterDirectiveEnvelope`、`ZoneDirective`、`Command` 或 Codable schema。
- 不改变命令执行链路、AI 决策、规则校验、日志写入、raw JSON 保存或玩家态/开发态分层。
- 不改变 `commandResultTitle(_:)`、`commandDisplayTitle(_:)` 或 `resultLine(_:)` 的可见文案逻辑。
- 不扩大到方面军令卡片、诏令朝议摘要块、CommandPanel 状态反馈或 DiplomacyPanel 复合行。
- 不声明完整 VoiceOver 实机验收、截图布局验收、全项目玩家可见残留清零或发布级 UI 完成。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成完整 AI 面板验收、结构化 event payload 或发布级 UI 验收完成。

## 遗留风险

- `AgentPanelView` 的方面军令卡片和诏令朝议摘要块仍待后续复合读屏硬化。
- `CommandPanelView` 状态/反馈行、`DiplomacyPanelView` 归附记录/诸国/关系行仍待后续小切片。
- 完整 VoiceOver 实机、截图布局、全项目玩家可见残留扫描、真 LLM 输出本地化和发布级 UI 验收仍未完成。
