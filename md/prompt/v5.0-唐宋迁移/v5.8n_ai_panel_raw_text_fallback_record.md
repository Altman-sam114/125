# v5.8n AI 面板原始文本兜底硬化记录

完成日期：2026-07-06

## 目标

- 继续收口唐宋默认 AI 面板中的玩家可见英文、Latin、JSON、schema key、raw id 和 ASCII 分隔符。
- 覆盖旧存档、fallback 记录或失败路径把 `marshal directives`、`Compiled zone directive(s)`、diagnostics、errors、raw JSON 等原始文本带到玩家态的风险。
- 保持本轮为 `AgentPanelView` 只读显示桥小切片，不改变 AI 记录、命令、规则或 schema。

## 并发子 Agent 发现

- `preferredFrontZoneId` 的重点防区仍可直接调用外部 `zoneDisplayName`，默认闭包可能返回 raw `FrontZoneId`。
- `mandateIntent`、`courtPolicy`、`summary`、`strategicIntent` 和 `contextSummary` 仍可能直出英文 fallback、JSON 痕迹或 schema key。
- 唐宋“军令诊断”和“异常”仍可展开显示 diagnostics / errors 原文。
- 唐宋“军议原文”仍可展开 raw JSON。
- 方面军令摘要和目标列表仍使用 ASCII `/`、`, `。
- 未识别命令标题仍可能显示英文命令名或 raw 字段。

## 实现内容

- `preferredFrontZoneId` 显示改走 `displayName(for:)`，缺运行态防区名时唐宋路径显示“未命名方面”。
- 新增唐宋叙述文本兜底：`summary`、`strategicIntent`、`mandateIntent`、`courtPolicy` 和 `contextSummary` 若包含 Latin、JSON 痕迹、引号、下划线等 raw 文本风险，则降级为中文摘要。
- 未识别命令标题若包含 Latin、参数括号或 raw 文本风险，唐宋路径显示为“军令”。
- 唐宋 diagnostics、errors 和 raw JSON 不再展开原文，改为计数或“军议原文已记录”提示；原始数据仍保留在既有记录结构中。
- 方面军令摘要连接符改为“·”，目标和解释字段州府列表改为“、”。

## 非目标

- 不改 `AgentDecisionRecord`、`WarDirectiveRecord`、`TheaterDirectiveExplanationSummary` 或 `TheaterDirectiveEnvelope`。
- 不改 raw JSON 存储、Codable schema、AI 决策算法、decoder、compiler、`WarCommandExecutor`、`RuleEngine` 或任何底层命令。
- 不把 UI 兜底写成 AI 输出已结构化本地化；完整真 LLM 输出本地化仍是后续任务。
- 不处理外交、战报、MapEditor、截图、布局或 VoiceOver 的完整发布级验收。

## 验证口径

- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 提交并推送 `origin/main` 后，由 GitHub Actions `WWIIHexV0 CI Results` 云端验证。
- Agent C 核对本轮最新 commit 对应 artifact 的 manifest、JUnit、主构建日志、failure summary 和 static checks。

## 遗留风险

- 原始 diagnostics、errors 和 raw JSON 仍按既有 schema 保存，供调试追溯；本轮只改变唐宋玩家态显示。
- 未来真 LLM 若输出混合中英文但语义可用，当前 Latin guard 会保守降级为中文摘要；更细粒度本地化可后续在 AI 输出层或结构化字段层处理。
- 完整结构化 event payload、全局 accessibility、截图/布局/VoiceOver 和发布级 UI 验收仍待后续 v5.8/v5.9 收口。
