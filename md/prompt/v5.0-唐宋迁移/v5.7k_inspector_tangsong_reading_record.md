# v5.7k 军队/州府检查面板唐宋读法记录

本文记录唐宋 v5.7k 小切片：补齐默认主游戏检查面板的唐宋显示桥，减少玩家选中军队或州府后看到的英文、二战资源缩写和 legacy 势力名残留。

## 1. 目标

- 唐宋场景下，军队详情面板显示军队、政权、指挥、地块、州府、动态方面、防区、粮道、兵力、退却口径、补给、状态和编成等读法。
- 唐宋场景下，州府详情面板显示州府详情、地块控制、控制政权、地形、城池、城级、关隘、粮草、围城、工坊、产出、方面、防区、前线压力、道路、目标、己方军队和可见敌军等读法。
- 唐宋场景下，产出读作丁口、钱帛、粮草；编成读作禁军、骑军、厢军、器械；围城摘要使用当前 `GameState.displayName(for:)` 的政权名。

## 2. 非目标

- 不改 `Division`、`RegionNode`、`ComponentType`、`EconomyResources`、`Faction` 或存档 Codable schema。
- 不改 `GameState`、`Command`、`CommandValidator`、`CommandExecutor`、`RuleEngine`、`WarCommandExecutor` 或 `VictoryRules`。
- 不改变 hex / region / theater / front / deploy 权威边界。
- 不新增真实多政权选择器、持久化配置、将领面板唐宋化或发布级布局验收。
- 不跑本机测试、构建、模拟器或截图验收；本轮按用户要求只走云端验证。

## 3. 实现链路

```text
RootGameView
  -> UnitInspectorView(isTangSongScenario, factionDisplayName)
  -> RegionInspectorView(isTangSongScenario, factionDisplayName)
  -> 只读显示唐宋字段与当前政权名
  -> 不提交命令，不写 GameState
```

`factionDisplayName` 由 `RootGameView` 传入 `container.gameState.displayName(for:)`，避免检查面板直接读取 `Faction.displayName` 导致唐宋主路径露出 `Germany` / `Allies`。
