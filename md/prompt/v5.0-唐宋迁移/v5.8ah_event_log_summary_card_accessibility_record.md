# v5.8ah 战报摘要卡片读屏硬化记录

## 目标

继续收口唐宋默认主路径中的战报面板，让“本回合战报 / 最近战报”摘要卡片在读屏时作为一条完整摘要朗读，减少标题、回合、汇总文字和重点条目被拆散朗读的问题。

## 实现范围

- `EventLogView`：
  - “本回合战报 / 最近战报”摘要卡片补充 `.accessibilityElement(children: .ignore)`。
  - 新增 `turnReportAccessibilityLabel(for:)`，唐宋路径读作“战报摘要：本回合战报 / 最近战报”。
  - 新增 `turnReportAccessibilityValue(for:)`，复用 `TurnReportSummary.turnText`、`summaryText` 和 highlights。
  - 摘要卡片视觉、`turnReportSummary` 派生、日志来源和 highlight 生成逻辑保持不变。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
  - 将 v5.8ag 最终文档提交 `97d97b4` 的云端通过结果补写到 `update_log.md`。

## 并发子 Agent

- UI 只读扫描确认 v5.8ag 只覆盖最近战报列表行；战报顶部胜负、评分估算、本回合摘要和胜利目标等复合卡片仍可继续做逐卡片读屏硬化。
- 文档扫描建议 v5.8ah 接续 v5.8ag 遗留项，并新增 `v5.8ah_event_log_summary_card_accessibility_record.md`。
- 本轮选择最窄且直接承接 v5.8ag 的“本回合战报 / 最近战报”摘要卡片，不扩大到胜负、评分估算或胜利目标卡片。

## 禁止项

- 不改变 `GameLogEntry`、`CommandResultLogEntry`、事件写入端、日志结构或 Codable schema。
- 不改变 `TurnReportSummary` 派生逻辑、`TangSongEventLogMessage` 显示桥语义或结构化 event payload。
- 不改变 `Command`、`CommandValidator`、`CommandExecutor`、`RuleEngine`、`WarCommandExecutor`、AI 决策或 hex/region/theater/front/deploy 权威边界。
- 不扩大到胜负卡片、评分估算卡片、胜利目标卡片、HUD 卡片、主棋盘 focus tree 或全项目 accessibility 扫描。
- 不声明完整 VoiceOver 实机验收、截图布局验收、全项目玩家可见残留清零或发布级 UI 完成。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成完整全项目写入端本地化、结构化 event payload 或发布级 UI 验收完成。

## 遗留风险

- 胜负摘要、评分估算、胜利目标、HUD 目标/下一步等复合区域仍未做逐卡片合并读屏审计。
- 完整 VoiceOver 实机、截图布局、全项目玩家可见残留扫描、真 LLM 输出本地化和发布级 UI 验收仍未完成。
