# v5.7g 下一步提示读取移动/攻击高亮数量记录

本文记录唐宋 v5.7g 小切片：在 v5.7a-f 已经补上下一步提示、统一目标锚点、目标州府 spotlight、每回合战报摘要和新局/指挥身份包装后，让 HUD “下一步”提示读取既有移动/攻击高亮数量，给玩家更具体的可行动项。

## 1. 目标

- 选中可行动宋军时，下一步提示能说明当前有多少可攻击目标。
- 选中可行动宋军时，下一步提示能说明当前有多少可行军格。
- 优先保留招降、围城、招抚、解围、修城等已有目标提示。
- 只复用既有 UI 高亮结果，不新增规则入口。

## 2. 非目标

- 不新增命令。
- 不提交 `Command`，不调用 `RuleEngine`。
- 不写 `GameState`、`eventLog`、hex / region / theater / front / deploy 或 diplomacy。
- 不新增 `CommandValidator` dry-run 合法性面板。
- 不实现完整教程系统、任务链、路线指引、自动镜头或发布级截图验收。
- 不改变 `MovementRules`、`WarRelationRules`、`CommandValidator`、`WarCommandExecutor` 或 `RuleEngine`。

## 3. 实现链路

```text
AppContainer.selectedActionDivision
  -> refreshHighlights()
  -> movementHighlights = MovementRules().movementRange(...)
  -> attackHighlights = 敌对关系 + 射程内敌军坐标

RootGameView.nextActionHint
  -> 保留胜负 / 观战 / 阶段 / 选军 / 已行动 / 围城外交目标提示
  -> 读取 attackHighlights.count 与 movementHighlights.count
  -> HUD 下一步显示可攻击目标数与可行军格数
```

真实移动、攻击、围城和外交仍由玩家点击按钮或地图后进入 `Command -> RuleEngine -> CommandValidator -> CommandExecutor`。
