# v5.8v MapEditor 错误与输入可访问性硬化记录

## 目标

继续收口 MapEditor 在唐宋默认路径下的玩家可见与读屏可见残留，重点处理默认资源读取/覆盖/预览失败时的 raw 错误直出，以及新建输入框把示例文字误写入真实草案的风险。

## 实现范围

- `MapEditorViewModel`：
  - 新增错误显示包装，保留 `MapEditorExportError` 和 `MapEditorGameResourceBridgeError` 的中文说明。
  - 读取默认唐宋资源、覆盖默认唐宋资源、生成资源预览失败时不再使用 `String(describing: error)` 作为玩家文案。
  - 新建州府、方面、军队名称默认改为空字符串，继续由自动命名 fallback 生成“州府 N / 方面 N / 军 N”。
- `MapEditorView`：
  - 新建州府、方面、军队输入框补上下文 accessibility label/hint。
  - 选中地块州府/方面编辑框补上下文 accessibility label/hint。
  - 错误区显示和 accessibility label 统一加“错误：”前缀。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 禁止项

- 不改变导出 JSON schema、默认资源文件名、默认资源路径或 MapEditor 存储结构。
- 不改变 `Faction.allies/germany`、`GamePhase.alliedPlayer`、`RegionId`、`TheaterId`、主游戏规则、AI 决策或 Codable raw 值。
- 不做完整 VoiceOver 实机验收、截图验收、横竖屏布局验收或发布级 UI 验收。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 并发子 Agent

- MapEditor 只读扫描子 Agent 已返回，定位到 `MapEditorViewModel` 三处 raw `String(describing:)`、MapEditor 错误区缺少错误前缀、新建输入默认值和信息面板编辑框上下文弱的问题。
- 文档扫描子 Agent 因并发额度断开，未产出可用结果；文档更新由主线程按现有 v5.8u 记录和本轮真实 diff 完成。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成发布级 UI 验收完成。

## 遗留风险

- 主棋盘逐地块/军队/攻击目标 VoiceOver 可聚焦与可操作路径仍未实现。
- 完整截图、iPhone/iPad 横竖屏布局、完整 VoiceOver 实机和 v5.9 发布验收仍待后续。
- 结构化 event payload、全项目写入端本地化和真 LLM 输出本地化仍未完成。
