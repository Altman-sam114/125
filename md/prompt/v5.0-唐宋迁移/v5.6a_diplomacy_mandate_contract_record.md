# v5.6a 外交归附与天命规则合同首轮记录

本文记录 v5.6 的第一个规则层切片。目标是先把“招抚/归附/天命”从上游解释字段推进为可校验、可执行、可记录的 `Command` 合同；它不是完整外交 UI、完整多政权战争关系、治理政策或唐宋胜利规则。

## 1. 目标

- 新增最小 `MandateState`，记录 faction 级天命/合法性分数。
- 扩展 `DiplomacyState`，让国家级关系支持 `tributary`、`submitting`、`negotiating`，并保存 `PacificationRecord`。
- 新增 `Command.proposeSubmission(negotiatorId:targetCountryId:targetRegionIds:)`。
- 归附提议必须经 `CommandValidator -> CommandExecutor -> RuleEngine`，不能由 UI、AI 或解释字段直接修改 `GameState`。
- `DataLoader` 在唐宋默认剧本中初始化宋/割据的天命分数。

## 2. 非目标

- 不把 `TheaterDirectiveEnvelope.pacificationTargets` 直接编译成归附命令。
- 不新增外交面板写操作或 UI 按钮。
- 不改变 `TurnOrderState.relations` 的 `.allies/.germany` 全局战争关系。
- 不把吴越等单个国家的关系投射成 tactical neutral；当前战术敌我仍由 legacy `Faction` 和 `WarRelationRules` 决定。
- 不交割 hex / region controller，不转换或删除部队，不调用 `StrategicStateSynchronizer`。
- 不实现 `GovernancePolicy`、民心、治安、税粮、叛乱、完整天命胜利规则或发布级外交玩法。

## 3. 规则合同

新增底层命令：

```text
Command.proposeSubmission(
  negotiatorId: String,
  targetCountryId: CountryId,
  targetRegionIds: [RegionId]
)
```

校验条件首轮如下：

- 当前 phase 必须允许命令，谈判军队必须属于当前 active faction，且未行动、未撤退、可行动。
- 谈判方必须能在 `DiplomacyState` 中找到本 faction 的 primary country。
- 目标国家必须存在，且不能与谈判军队同 faction。
- 谈判方天命必须达到最小阈值。
- 谈判方与目标国家必须已有可招抚关系：`neutral`、`hostile`、`atWar` 或 `negotiating`。
- `targetRegionIds` 不能为空；若目标国家有 capital region，则本轮必须包含其 capital region。
- 目标州府必须存在，且当前 controller 仍为目标国家的 legacy faction。
- `neutral` / `negotiating` 目标可以因低 warSupport 进入招抚；`hostile` / `atWar` 目标需要目标州府已有围城压力、城防归零且守军不再 supplied。

执行结果首轮如下：

- 目标关系写为 `DiplomaticStatus.submitting`。
- 写入 `PacificationRecord`，记录 turn、actor country、target country、target regions、previous/result status、mandate delta 和原因。
- 调整谈判 faction 的 `MandateState.legitimacyByFaction`。
- 标记谈判军队已行动，并写入 diplomacy 分类日志。
- 不改变地图控制权、部队归属或 `TurnOrderState.relations`。

## 4. 当前架构边界

唐宋默认剧本仍通过 legacy 二元 `Faction.allies` / `Faction.germany` 承载战斗、控制权和补给敌我判断；`DiplomacyState` 是国家级投影，`TurnOrderState.relations` 是当前战争合法性的 power 级权威。因为吴越、南唐、后蜀、北汉等当前仍桥接到 `.germany`，v5.6a 不能把一个国家的归附写成 `.allies/.germany` 和平，否则会让所有割据政权同时失去战术敌对关系。

因此本轮只建立规则合同和状态记录。后续若要实现单国 tactical neutral、完整纳土交割或多政权开战，必须先把地图控制权、单位归属或 power/country 归属继续从二元 `Faction` 迁移到更细粒度的 `PowerId` / `CountryId`。

- v5.6 后续需要只读 UI 展示 `MandateState` 与 `PacificationRecord`，但 UI 不能直接写状态。
