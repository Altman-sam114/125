# v5.8l 将领计划摘要与固定英文 UI 硬化记录

完成日期：2026-07-06

## 目标

- 继续收口唐宋默认玩家主路径中的 raw id 与固定英文。
- 优先处理将领“已拟军令”摘要、SpriteKit 空棋盘、macOS 菜单和通用信息按钮。
- 保持本轮为 UI 显示桥小切片，不改变规则、计划记录 schema、地图状态或 app 生命周期。

## 并发子 Agent 发现

- `GeneralCommandPanelView.operationSummary` 在唐宋路径仍拼接 `targetRegionId/sourceRegionId/zoneId.rawValue`。
- `BoardScene` 空棋盘标题为 `Hex Board`。
- `WWIIHexV0MacApp` 菜单仍为 `Game / End Turn / New Game`。
- `InfoPanelToggle` 固定显示 `[ INFO ]`，也会影响可访问读法。

## 实现内容

- `GeneralCommandPanelView` 新增 `regionDisplayName` / `zoneDisplayName` 只读查找闭包。
- `RootGameView` 在两个 `GeneralCommandPanelView` 调用处传入运行态州府和方面名称。
- 唐宋“已拟军令”优先显示目标州府、来源州府或方面名称；缺名时显示“未命名州府 / 未命名方面”。
- `BoardScene` 空棋盘标题改为“舆图加载中”。
- macOS 菜单改为“军务 / 结束回合 / 重新开局”。
- `InfoPanelToggle` 显示改为“详情”，并补 `.accessibilityLabel("详情")`。

## 非目标

- 不改 `PlayerPlannedOperation`、`PlayerCommandState` 或 Codable schema。
- 不改将领命令生成、`ZoneDirective`、`WarCommandExecutor`、`Command`、`RuleEngine` 或 AI 决策。
- 不改地图渲染状态、MapDisplayAdapter、SpriteKit 图层逻辑或 macOS app 生命周期。
- 不处理 AI 面板 raw JSON/diagnostics、外交面板 Latin guard 或全部数据名清洗。

## 验证口径

- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 提交并推送 `origin/main` 后，由 GitHub Actions `WWIIHexV0 CI Results` 云端验证。
- Agent C 核对本轮最新 commit 对应 artifact 的 manifest、JUnit、主构建日志、failure summary 和 static checks。

## 遗留风险

- `GeneralCommandPanelView` 的 `zone.name`、`division.name`、`targetRegion.name` 仍直接来自运行态数据；默认唐宋数据为中文，但未来混入 legacy/英文数据时仍需更完整的名称清洗。
- AI 面板解释文本、diagnostics/raw JSON 和外交面板国家/集团名仍有后续硬化空间。
