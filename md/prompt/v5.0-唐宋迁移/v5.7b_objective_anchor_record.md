# v5.7b 首屏统一目标锚点记录

本文记录唐宋 v5.7b 小切片：在 v5.6i 已能显示胜利目标进度、v5.7a 已有“下一步”提示的基础上，把主要宋统一目标拆成“已据/待取”关键州府列表，放到 HUD 首屏。

## 1. 目标

- 在唐宋场景 HUD 中显示紧凑的“目标”提示。
- 复用现有 `VictoryRules.objectiveProgress(in:)` 的主要 `majorVictory` 条件，不另建规则。
- 用 `MapState.controllerOfObjective(named:)` 判断关键州府当前控制方，列出已由宋控制和仍待夺取的目标。
- 让玩家看到 `统一进度 2/4` 时，也能立刻知道对应目标是开封、洛阳、太原、金陵、成都、杭州等州府。

## 2. 非目标

- 不新增胜利条件，不改变 `VictoryRules.updateVictoryState(in:)` 或 `VictoryRules.objectiveProgress(in:)`。
- 不新增地图高亮层、目标点击定位、路线指引或任务链。
- 不提交 `Command`，不调用 `RuleEngine`，不写 `GameState`、`eventLog`、hex、region、theater、front 或 diplomacy。
- 不替代战报面板中的完整目标进度摘要。
- 不做截图、模拟器、Xcode build/test 或完整布局验收。

## 3. 实现链路

```text
HUDView.objectiveGuideText
  -> VictoryRules.objectiveProgress(in: gameState)
  -> primary majorVictory progress
  -> MapState.controllerOfObjective(named:)
  -> 已据 / 待取关键州府短列表
  -> HUD “目标”只读提示
```

真实胜负仍由 `VictoryRules.updateVictoryState(in:)` 判定；真实行动仍由玩家点击命令按钮后进入 `Command -> RuleEngine`。

## 5. 并发审计

本轮按用户要求继续使用并发子 Agent：

- UI 子 agent 只读确认首屏 HUD 是目标锚点最小接入口；后续若要点击目标定位，可复用 `AppContainer` 选中 hex / region 链路。
- 数据/规则子 agent 只读确认唐宋 960 JSON 已有 `victoryConditions` 和 objective id，可支持只读目标名单，不需要新增数据 schema。

主线程采用 HUD 只读文字锚点方案，避免提前改 SpriteKit 高亮层或选区状态。
