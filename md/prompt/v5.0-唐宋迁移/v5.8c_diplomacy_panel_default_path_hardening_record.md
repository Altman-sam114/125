# v5.8c 外交面板默认主路径读法硬化记录

## 目标

- 延续 v5.8 发布候选硬化，把唐宋默认外交面板中容易暴露英文状态、raw id 或二战兼容口径的读法收口为玩家可读的唐宋文案。
- 只处理 `DiplomacyPanelView` 的只读显示桥，让天命、诸国、关系、君主朝议和归附记录更贴近 `jianlong_960_unification` 默认主路径。
- 通过 `RootGameView` 向外交面板注入运行态州府和方面防区名称查找，避免归附目标和君主重点方面默认显示 raw id。

## 非目标

- 不改变 `DiplomacyState`、`MandateState`、`CountryProfile`、`DiplomaticRelation` 或 `PacificationRecord` 的 Codable schema。
- 不改变 `Command.proposeSubmission`、`CommandValidator`、`CommandExecutor`、`RuleEngine` 或 `TurnManager` 的外交执行语义。
- 不改变 `TurnOrderState.relations`、`WarRelationRules.canTarget`、战术敌我、外交投影、天命胜利规则或任何控制权。
- 不交割 hex/region 控制权，不转换部队，不实现真实单国 tactical neutral、完整 annexation、治理政策或外交纳土结算。
- 不处理 EventLog 原始日志、MapEditor 英文错误、README legacy 段落、全局 accessibility、截图、布局或 VoiceOver 实机验收。

## 实现说明

- `DiplomacyPanelView` 新增 `regionDisplayName` 与 `zoneDisplayName` 只读闭包，默认保留 legacy raw id fallback，唐宋场景由 `RootGameView` 注入运行态州府和防区名称。
- 国家副标题在唐宋场景显示政权与集团名；集团缺失时显示“未命名集团”，国家缺失时显示“未知政权”。
- `DiplomaticStatus` 在唐宋场景显示为盟好、称臣、协战、中立、敌对、交战、归附中或议和。
- `RulerStrategicPosture` 在唐宋场景显示为进取、固守、维系诸国或安定边面；君主主事 fallback 显示为君主诏令、割据国主或国主议事。
- 归附记录目标州府优先显示运行态州府名，缺失时唐宋路径显示“未命名州府”；重点方面缺失时显示“未命名方面”。
- 唐宋场景下不直接展示上游 `record.rationale`，改用本地摘要“朝议：...”，避免默认主路径铺开英文或 raw id。

## 并发子 Agent 结果

- 代码审计子 Agent 只读扫描 `DiplomacyPanelView`，指出外交面板仍有 `blocId.rawValue`、`DiplomaticStatus.displayName` 英文、`rulerAgentId`、`RulerStrategicPosture.displayName` 英文、`zoneId.rawValue`、`countryId.rawValue`、`targetRegionIds.rawValue`、`record.rationale` 和 faction fallback 等默认主路径残留。
- 文档审计子 Agent 只读给出同步清单：更新 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/flow/01_overall_core_flow.mermaid`、`update_log.md`，并新增本记录文件。
- 主 Agent 整合时确认本轮只触碰外交面板显示桥和文档，不改规则层、外交状态层、关系投影层或地图控制权。

## 验证口径

- 按用户当前要求，本轮不在本机运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 本轮完成后直接 commit 并 push 到 `origin/main`，触发 GitHub Actions `WWIIHexV0 CI Results`。
- Agent C 验收必须下载最新 `origin/main` commit 对应的未加密 artifact，核对 `ci-artifact-manifest.json` 的 `branch`、`commitSha`、`runId`、`runAttempt`，并阅读 `junit.xml`、`static-checks.log`、`xcodebuild.log` 和 `ci-failure-summary.md`。
- 如果云端失败，默认在 `main` 上追加修复 commit，再 push 触发新 run；不做回滚式处理。

## 遗留风险

- EventLog 原始日志仍可能显示上游英文或 raw id，需要后续 v5.8 RC 残留扫描继续处理。
- MapEditor 英文错误和工具侧 legacy 文案仍未系统收口。
- README 中仍保留大量 legacy 历史说明，后续发布候选需要区分历史记录与默认主路径文案。
- 未做截图、布局、VoiceOver、Dynamic Type 或小屏实机验收。
- 当前外交仍不是完整多国战术中立、完整 annexation、治理政策或外交纳土结算系统；v5.8c 只改善默认外交面板读法。
