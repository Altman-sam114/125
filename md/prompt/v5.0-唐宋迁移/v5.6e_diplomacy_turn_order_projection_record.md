# v5.6e 外交关系到 Power 战争关系保守投影记录

本文记录唐宋 v5.6e 小切片：把 `DiplomacyState` 的国家级外交关系保守投影回 `TurnOrderState.relations`，让 `WarRelationRules.canTarget` 读取的 power 级战争合法性不再长期脱离归附记录。

## 1. 目标

- `Command.proposeSubmission` 成功写入国家关系后，同步一次 legacy power 级关系。
- 只要宋与割据阵营之间仍有任一国家关系是 `hostile` / `atWar`，`.allies/.germany` 仍保持 `atWar`。
- 避免吴越等单国进入 `submitting` 后，让所有割据势力提前不可攻击。
- 保持攻击、补给通行、围城合法性继续通过 `WarRelationRules.canTarget` 读取 `TurnOrderState.relations`。

## 2. 实现口径

```text
diplomacyState.relations
  -> projectedPowerRelationStatus(between: .allies, and: .germany)
  -> if any cross-faction country relation is hostile/atWar: .atWar
  -> else map all tributary/allied/coBelligerent to tributary/allied
  -> else submitting / negotiating / neutral
  -> TurnOrderState.setRelationStatus
```

`CommandExecutor.executeProposeSubmission` 在 `setRelationStatus` 成功写入国家关系后调用同步 helper。同步只影响 `TurnOrderState.relations`，不直接修改地图、单位、州府控制权或 theater/front/deploy。

## 3. 非目标

- 不实现吴越、南唐、后蜀、北汉等单国 tactical neutral。
- 不新增国家级 `PowerId`，不拆分 legacy `.allies/.germany` 部队归属。
- 不交割 hex / region controller，不转换、删除或收编部队。
- 不让 AI、UI 或事件绕过 `Command -> RuleEngine` 直接改 `GameState`。
- 不实现治理政策、民心、治安、税粮或叛乱。

## 6. 遗留风险

- 该同步仍是 `.allies/.germany` power 级聚合，不能表达单个国家与宋停战、其他割据国家继续战争的战术合法性。
- 当前部队和 hex controller 仍按 legacy faction 归属；归附后纳土、部队转属和战略层刷新仍需后续小切片。
- `Command.proposeSubmission` 目前只把目标关系推进为 `submitting`，没有完整称臣、纳土或治理后果。
