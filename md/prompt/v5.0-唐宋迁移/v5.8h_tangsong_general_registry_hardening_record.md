# v5.8h 唐宋将领注册表默认路径硬化记录

## 目标

- 延续 v5.8 发布候选硬化，消除唐宋默认主路径将领面板读取 legacy `generals.json` 的玩家可见残留。
- 复用已进入资源包的 `tangsong_characters.json`，把唐宋人物映射为兼容 `GeneralRegistry`，不新增底层规则 schema。
- 保留 `generals.json` 给显式 legacy 阿登和历史回归参考。

## 非目标

- 不删除 `generals.json`、阿登 region seed、旧将领测试夹具或 legacy Agent D 参考。
- 不重命名 `GeneralData`、`GeneralAssignment`、`Division`、`Faction.allies/germany` 或 Codable schema。
- 不改 `WarCommandExecutor`、`RuleEngine`、`ZoneDirective`、AI 决策记录或命令执行管线。
- 不在本轮处理命令面板 raw validation、inspector raw id、EventLog relatedRecordId、完整 accessibility 或截图/布局验收。
- 不做本机测试、build、Swift parse、Markdown 检查或 `git diff --check`。

## 实现说明

- 新增 `TangSongCharacterCatalogDefinition` / `TangSongCharacterDefinition`，将 `tangsong_characters.json` 的 id、name、role、powerId、commandStyle、traits 映射为兼容 `GeneralData`。
- `DataLoader.loadGeneralRegistry(for:)` 在 `jianlong_960_unification` 下读取 `tangsong_characters.json`；其他 scenario 或 nil 仍读取 legacy `generals.json`。
- `DataLoader.assignGenerals` 与 `AppContainer.bootstrap()` 都按当前 scenario 加载 registry，避免唐宋默认方面从 legacy 将领池取蒙哥马利、古德里安等人物。
- `GeneralCommandPanelView` / `GeneralProfileView` 在唐宋场景下对 rank、biography、skills 增加中文显示桥；若后续遇到 legacy 英文数据，显示“禁军都部署 / 方面都部署”“筹攻 / 转运 / 守城 / 突进 / 统军 / 军务”等兜底。
- `GeneralProfileView` 唐宋头像 accessibility label 改为中文“头像占位”。

## 并发子 Agent 结果

- 将领 explorer 只读确认 `DataLoader.loadGeneralRegistry()` 固定读取 `generals.json`，`DataLoader.assignGenerals` 与 `AppContainer.bootstrap()` 会让唐宋默认主路径使用 legacy registry；本轮已处理。
- 同一 explorer 指出 `GeneralCommandPanelView` / `GeneralProfileView` 直接显示 legacy rank、bio、skills；本轮已增加唐宋显示桥。
- UI/raw 文案 explorer 只读指出 `CommandPanelView.lastCommandMessage`、RuleEngine validation raw key、EventLog `relatedRecordId`、Unit/Region inspector raw id 和部分 accessibility 仍需后续收口；本轮只记录为后续风险，不混入当前数据切片。

## 验证口径

- 按用户当前要求，本轮不在本机运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 本轮完成后直接 commit 并 push 到 `origin/main`，触发 GitHub Actions `WWIIHexV0 CI Results`。
- Agent C / 主线程验收必须下载最新 `origin/main` commit 对应的未加密 artifact，核对 `ci-artifact-manifest.json` 的 `branch`、`commitSha`、`runId`、`runAttempt`，并阅读 `junit.xml`、`static-checks.log`、`xcodebuild.log` 和 `ci-failure-summary.md`。

## 遗留风险

- `tangsong_characters.json` 仍是轻量人物表，未包含完整唐宋将领偏好、region seed 或更细的人物传记。
- 命令面板 raw validation、EventLog relatedRecordId、Unit/Region inspector raw id、MapDisplayAdapter objectiveStatus 和完整 VoiceOver/截图验收仍待后续 v5.8/v5.9 小切片。
