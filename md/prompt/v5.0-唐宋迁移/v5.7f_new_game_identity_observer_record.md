# v5.7f 新局/重置包装与当前指挥身份/观战口径记录

本文记录唐宋 v5.7f 小切片：在 v5.7a-e 已经补上下一步提示、统一目标锚点、目标州府 spotlight 和每回合战报摘要后，为 HUD 增加当前指挥身份/观战模式口径，并为唐宋“重开剧本”增加确认。

## 1. 目标

- HUD 在唐宋场景显示当前“指挥 / 模式”短状态。
- 玩家能看出当前是宋可下令、宋待命、观战各方、玩家亲征或只读观战。
- 唐宋“新局”入口改为“重开剧本”，并在调用 `resetGame()` 前弹出确认。
- 继续保持第一屏紧凑，不做大段说明页。

## 2. 非目标

- 不实现真实多势力选择器。
- 不改变 `AppContainer.playerFaction`，当前玩家仍桥接为 `.allies` / 宋。
- 不新增 AI-only 配置页、存档槽、存档读写或剧本选择器。
- 不改变 `AppContainer.resetGame()` 的状态重置语义。
- 不改变 `GameState` schema、`RuleEngine`、`CommandValidator`、`CommandExecutor`、`TurnManager`、`Command` 或 `ZoneDirective`。
- 不做截图、模拟器、Xcode build/test 或本地轻量检查；本轮按用户要求只走云端 GitHub Actions 验证。

## 3. 实现链路

```text
RootGameView
  -> HUDView(gameState, playerFaction, observerModeEnabled)
  -> HUDView.commandIdentityText / commandModeText
  -> 显示宋可下令、宋待命、观战各方、玩家亲征或只读观战

NewGameButton
  -> 唐宋场景显示“重开剧本”
  -> RootGameView.confirmationDialog
  -> 用户确认后调用 AppContainer.resetGame()
  -> resetGame 仍重载 DataLoader.loadInitialGameState()
```

该切片只包装 UI 入口和当前身份读法，不改变命令执行、AI 推进、胜负判定或存档。
