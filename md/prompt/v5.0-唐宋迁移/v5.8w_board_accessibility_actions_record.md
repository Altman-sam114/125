# v5.8w 主棋盘 VoiceOver 自定义动作记录

日期：2026-07-06

## 目标

- 在不改 SpriteKit 渲染、不改规则系统、不改地图权威状态的前提下，补强主棋盘 `BoardSceneView` 的读屏可操作入口。
- 让 VoiceOver 用户能从棋盘元素直接触发下一处红色攻击目标、下一处高亮行军格或打开信息面板。
- 顺手补强 v5.0 总提示词的当前 main 直推、GitHub Actions artifact、v5.8 小切片和文档同步口径。

## 非目标

- 不实现逐 hex / 逐军队的独立 focus tree。
- 不改变 `BoardScene`、`HexNode`、`UnitNode` 或 SpriteKit 命中测试。
- 不改变 `Command`、`RuleEngine`、`WarCommandExecutor`、移动、攻击、围城、补给或胜负规则。
- 不把 `.allies/.germany` legacy 桥迁成真实多政权 `PowerId` schema。
- 不做截图、模拟器、VoiceOver 实机或 Xcode build/test。

## 改动

- `RootGameView.boardView` 增加三个 accessibility custom actions：
  - 攻击下一处红色目标。
  - 行军到下一处高亮地块。
  - 打开信息面板。
- `boardAccessibilityValue` 在唐宋场景下补充：
  - 当前地块中文列/行读法。
  - 州府名。
  - 控制政权。
  - 当前已选军队。
  - 红色攻击目标数量和高亮行军格数量。
- 攻击/行军 custom action 只复用现有 `AppContainer.handleBoardTap(_:)`，仍由既有点击链路提交攻击或移动命令，真实合法性继续由 `CommandValidator` / `RuleEngine` 决定。
- `codex-v5.0-唐宋aiagent历史策略迁移总提示词.md` 补充当前入口文档优先、并发边界、v5.8 小切片、main push、GitHub Actions artifact、Agent C manifest 核对、文档同步和每轮验收口径。
