# v5.8ab MapEditor 画布符号硬化记录

## 目标

继续收口 MapEditor 画布内玩家可见短字符号，让粮源和军队模板标记更贴近唐宋编辑器口径，并避免保留无用 SpriteKit 调试命名。

## 实现范围

- `MapEditorCanvasScene`：
  - 粮源画布标记从“补”改为“粮”。
  - 军队模板画布标记从单字 `禁/骑/弩/械/守/州/军` 改为“禁军、骑军、弩兵、器械、守军、州军、军队”。
  - 军队标记底板略放宽，以容纳两字标签。
  - 底图 SpriteKit 节点不再写 `name = "底图"`。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 并发子 Agent

- MapEditor 画布只读扫描确认 `MapEditorCanvasScene` 内实际可见文字集中在粮源标记和单位模板标记；未发现坐标、raw id 或 `node.name` 调试命名直接绘制到画布。
- 文档扫描确认 v5.8ab 需要同步 README、plan、flow、flowchart、update_log 和新增本阶段记录。
- 本轮只纳入 MapEditor SpriteKit 画布显示标签，不修改 SwiftUI 控件、编辑行为、快捷键处理或导出结构。

## 禁止项

- 不改变 `MapEditorDocument`、单位模板 id、导出 JSON schema、资源桥路径、`Faction.allies/germany`、`RegionId`、`TheaterId`、主游戏规则或 Codable raw 值。
- 不改变 `MapEditorCanvasScene` 鼠标事件、滚轮/平移/缩放逻辑、`handleShortcut(_:)` 或 macOS 键盘快捷键绑定。
- 不做完整 VoiceOver 实机验收、截图验收、macOS 编辑器布局验收、符号系统视觉验收或发布级 UI 验收。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成发布级 UI 验收完成。

## 遗留风险

- MapEditor 画布仍使用文字标签而非正式图标系统；如果后续需要发布级视觉，应单独设计图标/形状标记。
- MapEditor 真实 VoiceOver focus order、画布逐地块可访问路径、截图布局和 macOS 编辑器发布级体验仍未验收。
- 完整 v5.8 RC 玩家可见残留清单、资源授权、性能和 v5.9 发布验收仍待后续。
