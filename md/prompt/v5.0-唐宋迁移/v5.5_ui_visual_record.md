# v5.5 唐宋主界面术语桥与地图图层读法记录

本文记录 v5.5 的三个小切片：先把默认唐宋主界面里的 HUD、地图图层、信息面板、军令按钮和战报分类从二战/英文调试口径桥接到唐宋读法，再给 SpriteKit 地图、州府 overlay、前线/部署 overlay、道路/河流、围城圈、计划箭头和军队棋子加入唐宋场景视觉 token 与军旗符号，随后新增只读粮道虚线 overlay。它不是 v5.5 发布级 UI 完整收口；本轮没有接入外部美术资产、完整计划箭头系统，也没有做运行时截图或布局验收。

## 1. 目标

- 让默认 `jianlong_960_unification` 场景第一屏少显示玩家可见英文调试词。
- 保持 legacy 阿登路径原显示不变。
- 给 `MapDisplayLayer`、`GamePhase`、HUD、Command 面板和 EventLog 提供 `isTangSongScenario` 显示桥。
- 给 `TerrainStyle`、`HexNode`、`RegionOverlayNode`、`UnitNode`、`BoardScene` 和 `MapLayerOverlayNode` 提供 Tang Song 场景感知视觉桥。
- 让唐宋路径的棋子从 NATO APP-6 符号切到内置军旗轮廓和禁/骑/弩/械/守/军兵种字标。
- 让唐宋路径的地图可从可见友方军队读取 `SupplyRules.supplyRouteSummary`，绘制到最近可见粮源的只读粮道虚线。
- 只改变显示文案和只读分类，不改变 command、rule、log、Codable raw case、存档或 AI 执行权限。

## 2. 非目标

- 不重命名 `Faction`、`Division`、`ComponentType`、`ProductionKind`、`GamePhase`、`MapDisplayLayer` raw case。
- 不改变 `Command` / `ZoneDirective -> WarCommandExecutor -> RuleEngine` 统一执行管线。
- 不新增规则、AI 策略、外交归附、天命治理或胜利结算。
- 不接入外部图片、授权素材、asset catalog 或真实历史图像。
- 不改变 SpriteKit 布局架构、点击命中、camera、地图数据或可视性规则。
- 不把粮道虚线写成真实逐 hex 路径；当前只显示单位到最近粮源的抽象直线提示。
- 不运行本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。

## 4. 当前边界

- 本轮只是术语桥，不能宣称发布级 UI 已完成。
- 地图视觉仍沿用现有 SpriteKit 绘制体系，尚未建立纹理、图片资产、完整势力旗帜体系或授权素材清单。
- 粮道 overlay 复用 `SupplyRules.supplyRouteSummary` 的最近粮源和成本摘要；当前是只读抽象直线，不是逐 hex 路径，不改变补给判定。
- 未做运行截图、设备尺寸布局检查、Dynamic Type 极端字号检查或 VoiceOver 实测。
- legacy 阿登路径仍显示 Hex / Province / Event Log / Germany / Allies 等兼容读法。
