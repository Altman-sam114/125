# v5.8ac 军议与方面军令反馈硬化记录

## 目标

继续收口唐宋默认主路径中的军议、方面军令和命令面板反馈，减少玩家可见的 ASCII 比例、半英文 legacy 摘要和偏技术的“原文”口径。

## 实现范围

- `AppContainer`：
  - 唐宋路径的方面军令部分执行摘要从 `N/M` 改为“已执行 N 道，未执行 M 道”。
- `CommandPanelView`：
  - 对旧式 `General order executed...` 反馈做完整中文兜底。
  - 能解析数量时显示已执行/未执行；解析失败时显示“方面军令已执行。”
- `AgentPanelView`：
  - 唐宋玩家态把“军议原文”改为“军议详文”。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 并发子 Agent

- 主游戏 UI 只读扫描指出 `AppContainer`、`CommandPanelView` 与 `AgentPanelView` 仍有低风险玩家可见反馈可继续收口。
- MapEditor/文档只读扫描指出 README、plan、flow 和 flowchart 存在少量旧版本口径；本轮同步修正当前状态、默认资源 fallback 和 MapEditor 军队标签描述。

## 禁止项

- 不改变 `Command`、`ZoneDirective`、`WarDirectiveRecord`、`AgentDecisionRecord.rawJSON`、`GameLogEntry`、命令执行、AI 决策、规则系统或 Codable schema。
- 不改变 `AppContainer` 的选择逻辑、将领军令执行结果判定、`acceptedCount/totalCount` 计算或 `WarCommandExecutor` 输出。
- 不展开 raw JSON，不删除开发态/legacy 调试信息。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成完整全项目写入端本地化或发布级 UI 验收完成。

## 遗留风险

- 仍可能存在较低频的 legacy 英文写入端、真 LLM 输出或旧存档 raw 文案，需要继续按 v5.8/v5.9 小切片扫描。
- 完整结构化 event payload、真实多政权 schema、真 LLM 输出本地化、截图布局和发布级 UI 验收仍未完成。
