# v5.8x 面板控件 accessibility 与 fallback 硬化记录

## 目标

在 v5.8w 主棋盘 VoiceOver custom actions 之后，继续补强主游戏面板级控件的读屏状态和唐宋缺名 fallback，减少“只听到控件名但不知道当前状态”以及 raw id 兜底暴露。

## 实现范围

- `RootGameView`：
  - 地图图层选择器补 accessibility label/value/hint。
  - 紧凑信息面板分页补 accessibility label/value/hint。
  - 外交面板和军议面板的唐宋州府/方面缺名 fallback 改为“未知州府 / 未命名方面”，非唐宋 legacy 路径保留 raw id。
- `GeneralCommandPanelView`：
  - 头像入口读屏名与“查看档案”按钮区分。
  - 查看档案按钮补明确 hint。
  - 固守防线、进攻州府按钮补可用/停用 value 和停用原因 hint。
- 文档：
  - 同步 `README.md`、`md/plan/plan.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。

## 并发子 Agent

- MapEditor/SpriteKit/RootGameView 只读扫描指出图层 Picker、外交/军议面板 fallback、MapEditor 底图与画布 accessibility 仍可继续硬化。
- UI 面板只读扫描指出将领档案入口重复读屏、Unit/Region inspector 英文 legacy 标签和 UnitTooltip accessibility value 仍可继续硬化。
- 本轮只纳入图层/紧凑面板/将领按钮/fallback 小切片，避免混入英文 legacy 标签和 MapEditor 底图下一轮候选。

## 禁止项

- 不改变棋盘 custom actions、SpriteKit 渲染、地图命中测试、命令执行、将领计划、外交/AI 记录、JSON schema、规则系统或 legacy `Faction` 桥。
- 不实现逐 hex / 逐军队 focus tree。
- 不做完整 VoiceOver 实机验收、截图验收、横竖屏布局验收或发布级 UI 验收。
- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查、`git diff --check`、模拟器或截图。

## 验收口径

- 本地只允许读文件、审查 diff/status、commit/push。
- 推送 `origin/main` 后由 GitHub Actions 执行云端 build/static checks，并由 artifact manifest、JUnit、主构建日志和失败摘要核对结果。
- 未跑本机重测试、模拟器、截图和 VoiceOver 实机验收；不得把本轮写成发布级 UI 验收完成。

## 遗留风险

- Unit/Region inspector 英文 legacy 标签、MapEditor 底图按钮/画布 value 和 UnitTooltip accessibility value 仍可拆成后续小切片。
- 逐地块/逐军队 VoiceOver focus order、完整截图、iPhone/iPad 横竖屏布局、完整 VoiceOver 实机和 v5.9 发布验收仍待后续。
- 结构化 event payload、全项目写入端本地化和真 LLM 输出本地化仍未完成。
