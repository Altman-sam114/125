# v5.8p 兵力、粮道与数值标记 ASCII UI 硬化记录

完成日期：2026-07-06

## 目标

- 继续收口唐宋默认玩家路径中兵力、粮道、城防、评分、回合、军备成本和地图数值标签里的 ASCII 分隔符。
- 优先处理低耦合 UI / SpriteKit 显示桥，不改变规则、命令、AI、数据结构或事件写入职责。
- 延续 v5.8o 的小切片节奏，把玩家能直接看到的 `/`、`|`、` - `、`q,r` 和列表分隔逐步改成中文读法。

## 并发子 Agent 发现

- `EconomyPanelView` 唐宋军备成本行仍用 ASCII `|`。
- `UnitInspectorView` 唐宋军队详情的地块坐标、兵力、粮道成本、编成列表和多状态列表仍有 `q,r`、`/`、`, ` 或 ` / `。
- `UnitTooltipView` 与 tooltip accessibility label 的兵力仍使用 ASCII `/`。
- `GeneralProfileView` 辖下军队兵力仍使用 ASCII `/`。
- `UnitNode` 棋子标记中的兵力仍使用 ASCII `/`。
- `MapDisplayAdapter` 地图围城城防和粮道 overlay 标签仍使用 ASCII `/`。
- `HUDView` 回合进度、`EventLogView` 结算评分、metadata 和方面军议 highlight 仍使用 ASCII `/`、` - ` 或 ` / `。

## 实现内容

- `EconomyPanelView` 唐宋军备成本行改为“耗 ...，需 N 回合”。
- `UnitInspectorView` 唐宋军队详情地块坐标改为“第 q 列，第 r 行”，兵力和粮道成本改用“／”，编成和多状态列表改用“、”。
- `UnitTooltipView` 唐宋兵力显示与 accessibility label 改用“／”。
- `GeneralProfileView` 唐宋辖下军队兵力改用“／”。
- `UnitNode` 唐宋棋子兵力标记改用“／”。
- `MapDisplayAdapter` 唐宋围城城防和粮道 overlay 标签改用“／”。
- `HUDView` 唐宋回合进度改用“／”。
- `EventLogView` 唐宋胜负评分改用“／”，战报 metadata 改用中文分号，方面军议 highlight 改用“：”。

## 非目标

- 不改 `Division`、`SupplyRouteSummary`、`SiegeOverlayState`、`EconomyResources` 或 Codable schema。
- 不改生产、补给、围城、胜负、命令、AI 决策、事件写入或规则。
- 不宣称全项目已无英文或 ASCII；本轮只覆盖兵力、粮道、城防、评分、回合、成本和地图数值标签的小切片。
- 不做完整 accessibility、VoiceOver 实机、截图、横竖屏布局、资源授权或 v5.9 发布验收。

## 验证口径

- 按用户要求，本机不运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 提交并推送 `origin/main` 后，由 GitHub Actions `WWIIHexV0 CI Results` 云端验证。
- Agent C 核对本轮最新 commit 对应 artifact 的 manifest、JUnit、主构建日志、failure summary 和 static checks。

## 遗留风险

- `AppContainer` 写入端仍有部分英文拒绝/选中日志，后续可作为 v5.8q 源头反馈小切片处理。
- MapEditor 工具面板、bundle display name、完整 VoiceOver 和截图布局验收仍未完成。
- 本轮显示桥不替代结构化 event payload，也不处理真实 LLM 输出本地化。
