# v5.7e 每回合战报摘要记录

本文记录唐宋 v5.7e 小切片：在 v5.7a-d 已经补上下一步提示、统一目标锚点、目标州府定位和地图 spotlight 后，为战报面板增加只读“每回合战报摘要”，帮助玩家快速理解本回合或最近回合发生了什么。

## 1. 目标

- 在唐宋场景的战报面板中显示“本回合战报”或“最近战报”。
- 只读汇总已有规则事件，按战斗、州府、围城、粮道、外交、前线、方面、退却、整补和合围等类别统计。
- 额外读取最近 `AgentDecisionRecord` 与 `WarDirectiveRecord`，展示军议和方面军令摘要。
- 日志列表继续展示 `container.displayEventLog`，摘要统计单独读取 `gameState.eventLog`，避免 UI 交互日志污染战报统计。
- 保持 v5.7 首屏导览轻量化，不做完整结算页。

## 2. 非目标

- 不新增战报事件源。
- 不写 `GameState.eventLog`，不把摘要保存为权威状态。
- 不改变 `RuleEngine`、`CommandValidator`、`CommandExecutor`、`TurnManager` 或 `VictoryRules`。
- 不新增 `Command`、`ZoneDirective` 或 AI 决策 schema。
- 不让摘要参与命令合法性、AI 决策、胜负判定、状态同步或存档。
- 不做完整胜利结算、评分、教程任务链、自动镜头、截图或布局验收。

## 3. 实现链路

```text
RootGameView
  -> EventLogView(entries: container.displayEventLog)
  -> EventLogView(summaryEntries: container.gameState.eventLog)
  -> EventLogView(agentDecisionRecord: container.lastAgentDecisionRecord)
  -> EventLogView(directiveRecords: container.gameState.warDirectiveRecords)
  -> EventLogView(currentTurn: container.gameState.turn)

EventLogView.turnReportSummary
  -> 按 currentTurn 过滤 GameLogEntry
  -> 当前回合无规则事件时 fallback 到最近有记录回合
  -> 统计 LogDisplayCategory 分类数量
  -> 追加 AgentDecisionRecord / WarDirectiveRecord 的军议摘要
  -> 战报面板只读显示
```

摘要是展示层派生数据。它不调用 `RuleEngine`，不提交 `Command`，不写 `GameState`，也不改变 `WarDirectiveRecord` 或 `AgentDecisionRecord` 的生成职责。

## 7. 遗留风险

- 高层军议只读取最近 `AgentDecisionRecord`；更早回合的高层摘要当前不持久化，只能从 `WarDirectiveRecord` 补充方面军令。
- 本轮只做战报面板摘要，不做完整胜利结算页、评分、治理评价或教程任务链。
- 未做小屏、Dynamic Type、VoiceOver、截图或真实设备布局验收。
- 摘要按现有 `GameLogEntry.category` 分类；如果后续新增更细日志类别，需要同步扩展 `LogDisplayCategory` 与摘要统计顺序。
