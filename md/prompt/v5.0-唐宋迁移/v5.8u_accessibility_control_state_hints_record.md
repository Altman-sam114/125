# v5.8u accessibility 控件状态提示硬化记录

完成日期：2026-07-07

## 目标

在不改变命令执行、经济规则、胜利目标、地图聚焦逻辑、SpriteKit 地图交互、focus order、hit target、布局或 Codable schema 的前提下，继续收口唐宋默认路径中 VoiceOver 难以理解按钮可用状态和停用原因的问题。

## 实现摘要

- `CommandPanelView` 的军令按钮增加 accessibility value/hint，读屏可听到“可用 / 停用”以及观战、阶段不对、未选军队、已行动或缺少目标等原因。
- `EconomyPanelView` 的军备按钮增加 accessibility value/hint，把费用、建造回合、观战模式、当前阶段不可下令和资源不足原因绑定到按钮读法。
- `RootGameView` 的亲征选择与观战切换增加 accessibility value/hint，说明切换亲征政权会清空选择/高亮，观战会停用军令和府库命令。
- `HUDView` 的统一目标锚点按钮增加“查看目标”可访问标签、已据/待取状态和地图聚焦提示。

## 架构边界

- Hex 仍是战术权威；Region / Theater / Front / Deploy 边界不变。
- 玩家、AI、聊天命令和 MockAI 仍必须通过 `Command` / `ZoneDirective -> WarCommandExecutor -> CommandValidator -> RuleEngine` 执行。
- 本轮不改变 `CommandPanelView` 按钮可用性条件，不改变 `EconomyRules`、`ProductionKind`、`VictoryRules`、`AppContainer.focusObjective(id:)`、地图点击、AI 决策或任何 Codable raw 值。
- 本轮不是完整 VoiceOver 实机验收，不实现逐 hex 可聚焦地图操作路径。
