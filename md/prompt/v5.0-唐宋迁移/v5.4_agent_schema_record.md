# v5.4 唐宋 AI Agent 分层与军议文案记录

本文记录 v5.4 的前三轮小切片：先把 AI 面板、方面军令和战术名显示迁到唐宋军议口径，再把 simulated marshal raw JSON 的默认主事、strategicIntent、summary 和 rationale 迁到宋枢密院/割据行营、州府与粮道口径，随后补上 `TheaterDirectiveEnvelope` 的可选解释字段。它不是 v5.4 完整收口；皇帝/朝廷/枢密/节度使/转运使/州府守臣/外交使者的完整 schema、真实多 Agent JSON、外交归附和治理决策仍留后续切片。

## 1. 目标

- 让唐宋默认场景下的 AI 面板从 `AI Decision` / `Zone Directives` / `guderian` 占位转为“军议”“方面军令”“枢密院”口径。
- 让 `DirectiveType`、`CommandCategory` 和 `TacticName` 保留既有 raw case 与 Codable schema，但提供 `displayName(isTangSongScenario:)` 显示桥。
- 让唐宋场景下的战术显示从 `standardAttack`、`blitzkrieg`、`pincerMovement`、`fireCoverage` 等 raw case 转为进军、骑军突进、合围、弓弩压制、死守城关等读法。
- 让 `WarDirectiveRecord` 在 AI 面板中继续显示 zone、战术、成功/拒绝数量、目标州府和 commander id，帮助解释围城、招降、固守等结果。
- 让 `CommandResultSummary.directiveCommand` 在唐宋场景下保存底层命令的唐宋显示名，避免军议结果继续显示 legacy 命令名。
- 让 `MarshalBattlefieldSummary` 携带 `scenarioId`，让 `SimulatedMarshalLLMClient` 在唐宋场景下输出中文军议 strategicIntent、summary 和 directive rationale，避免默认 raw JSON 继续展示 Rundstedt / Eisenhower 与英文 simulated marshal 文案。
- 让 `TheaterDirectiveEnvelope` 新增可选 `mandateIntent`、`courtPolicy`、`pacificationTargets`、`supplyPriorities` 解释字段，唐宋 simulated marshal 根据首都、围城、粮道优先和招抚候选摘要填充这些字段。

## 2. 非目标

- 不修改 `ZoneDirective` JSON schema。
- 不让 `mandateIntent`、`courtPolicy`、`pacificationTargets` 或 `supplyPriorities` 参与 `TheaterDirectiveCompiler`、`WarCommandExecutor` 或 `RuleEngine` 执行；它们只是上游解释字段。
- 不接入真实 LLM、网络模型、API key 或外部模型路径。
- 不新增皇帝、朝廷、转运使、外交使者等完整 Agent 类型。
- 不改变 `WarCommandExecutor` 的单位选择、围城/招降、移动、攻击或固守规则。
- 不让 AI、UI、军议文案直接修改 `GameState`、`HexTile.controller`、`Division.coord`、`hexToTheater` 或 `hexToFrontZone`。
- 不跑本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。

## 4. 当前边界

- `TacticName` 的 raw case 仍是英文，保持 JSON 向后兼容；本轮只改显示桥。
- `TheaterDirective` 的 simulated rationale 已在唐宋场景下改用中文军议口径；非唐宋场景继续保留 raw tactic 与英文 simulated marshal 文案。
- `AgentDecisionRecord` 的 legacy Agent D 记录仍可能显示旧 agent id；默认战争 AI 主路径已通过 `WarDirectiveRecord` 在 AI 面板展示唐宋军令读法。
- `TheaterDirectiveEnvelope` 已新增 mandate/court/pacification/supply 可选解释字段，decoder 会校验其中 region id 存在；compiler 和 executor 当前不读取这些字段。
- `pacificationTargets` 不是外交归附执行结果，`supplyPriorities` 不是补给状态改写；它们只用于 raw JSON 解释和后续 Agent 分层接口。
- v5.4 完整 Agent 分层、真实多 Agent JSON、统治者/外交使者 schema 仍待后续。
- 这不是 v5.5 发布级 UI 收口；没有做运行时截图、布局烟测或可访问性验收。
