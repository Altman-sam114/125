# v5.4 唐宋 AI Agent 分层与军议显示桥记录

本文记录 v5.4 的第一轮小切片：先把 AI 面板、方面军令和战术名显示迁到唐宋军议口径。它不是 v5.4 完整收口；皇帝/朝廷/枢密/节度使/转运使/州府守臣/外交使者的完整 schema、真实多 Agent JSON、外交归附和治理决策仍留后续切片。

## 1. 目标

- 让唐宋默认场景下的 AI 面板从 `AI Decision` / `Zone Directives` / `guderian` 占位转为“军议”“方面军令”“枢密院”口径。
- 让 `DirectiveType`、`CommandCategory` 和 `TacticName` 保留既有 raw case 与 Codable schema，但提供 `displayName(isTangSongScenario:)` 显示桥。
- 让唐宋场景下的战术显示从 `standardAttack`、`blitzkrieg`、`pincerMovement`、`fireCoverage` 等 raw case 转为进军、骑军突进、合围、弓弩压制、死守城关等读法。
- 让 `WarDirectiveRecord` 在 AI 面板中继续显示 zone、战术、成功/拒绝数量、目标州府和 commander id，帮助解释围城、招降、固守等结果。
- 让 `CommandResultSummary.directiveCommand` 在唐宋场景下保存底层命令的唐宋显示名，避免军议结果继续显示 legacy 命令名。

## 2. 非目标

- 不新增或修改 `TheaterDirectiveEnvelope` / `ZoneDirective` JSON schema。
- 不接入真实 LLM、网络模型、API key 或外部模型路径。
- 不新增皇帝、朝廷、转运使、外交使者等完整 Agent 类型。
- 不改变 `WarCommandExecutor` 的单位选择、围城/招降、移动、攻击或固守规则。
- 不让 AI、UI、军议文案直接修改 `GameState`、`HexTile.controller`、`Division.coord`、`hexToTheater` 或 `hexToFrontZone`。
- 不跑本机 Xcode build/test、模拟器、Probe、Smoke、Stage Regression、Dynamic Theater Regression、Full 或性能测试。

UI 读法：

```text
RootGameView
  -> AgentPanelView(isTangSongScenario: gameState.isTangSongScenario)

AgentPanelView
  唐宋场景：
    AI Decision      -> 军议
    Agent            -> 主事
    Provider         -> 来源
    Intent           -> 意图
    Context          -> 战况
    Zone Directives  -> 方面军令
    Command Results  -> 军令结果
    Errors           -> 异常
    Raw JSON         -> 原始 JSON

WarDirectiveRecord summary
  -> 指令类型 / 唐宋战术名 / 成功数与拒绝数 / 目标州府 id
  -> commanderAgentId 以“将令”单独展示
```

命令结果显示：

```text
CommandResultSummary.directiveCommand(..., isTangSongScenario:)
  false -> command.displayName
  true  -> command.displayName(isTangSongScenario: true)

TurnManager.executeDirectiveEnvelope
  -> 用 nextState.isTangSongScenario 保存 AI 军令结果显示名

AppContainer.submitPlayerDirective
  -> 用 startState.isTangSongScenario 保存玩家宏观将军命令结果显示名
```

## 4. 当前边界

- `TacticName` 的 raw case 仍是英文，保持 JSON 向后兼容；本轮只改显示桥。
- `TheaterDirective` 的 simulated rationale 仍可能包含 raw tactic；完整唐宋化 JSON rationale 留给后续 v5.4 切片。
- `AgentDecisionRecord` 的 legacy Agent D 记录仍可能显示旧 agent id；默认战争 AI 主路径已通过 `WarDirectiveRecord` 在 AI 面板展示唐宋军令读法。
- 这不是 v5.5 发布级 UI 收口；没有做运行时截图、布局烟测或可访问性验收。
