# v5.6h 胜负原因显示与战报摘要记录

本文记录唐宋 v5.6h 小切片：在 v5.6g 已能由 `VictoryRules` 写入 `VictoryState.winner / reason` 的基础上，把胜负原因以玩家可读方式显示到 HUD 与战报面板。

## 1. 目标

- `VictoryReason` 提供唐宋场景显示桥，保留 enum raw value 和 Codable 兼容。
- HUD 胜负栏从 `VictoryState.reason` 派生显示原因，例如“宋胜利：关键州府与天命达标”。
- 战报面板顶部只读显示当前胜负摘要，帮助玩家理解结算原因。
- 保持胜负权威仍来自 `VictoryRules.updateVictoryState`。

## 2. 非目标

- 不新增胜利条件、不改变 `VictoryRules` 判定语义。
- 不把 UI 或战报面板写成胜负权威。
- 不向 `eventLog` 写入派生胜负事件，避免把展示摘要误当规则结果。
- 不做完整胜利面板、结算页、评分档位、治理评价或单国胜负。

## 3. 实现链路

```text
VictoryRules.updateVictoryState
  -> VictoryState.winner / reason
  -> VictoryReason.displayName / eventMessage
  -> HUDView.victoryText
  -> EventLogView 只读胜负摘要
```

## 7. 遗留风险

- 当前只是 HUD 与战报面板中的简短原因文案，没有完整胜利结算页。
- 战报摘要是从当前 `VictoryState` 派生，不会成为可回放的历史事件；后续若需要正式结算记录，应设计独立 schema。
