# v5.8d 战报日志默认主路径读法硬化记录

## 目标

- 延续 v5.8 发布候选硬化，减少唐宋默认战报面板中英文命令、raw validation、交互日志和二战兼容口径的玩家可见暴露。
- 让 `EventLogView` 的战报正文和“本回合战报 / 最近战报”摘要 highlight 使用同一套唐宋显示桥。
- 保持日志写入端、规则执行端和 Codable schema 不变，先用 UI 层显示桥处理高频默认主路径残留。

## 非目标

- 不改变 `GameLogEntry.message`、`CommandResultLogEntry`、`CommandResult` 或任何 Codable schema。
- 不改变 `CommandValidator`、`CommandExecutor`、`RuleEngine`、`WarCommandExecutor`、`TurnManager` 或事件写入职责。
- 不新增结构化 event payload，不把日志显示桥作为规则来源。
- 不处理 MapEditor 英文错误、README/flow 标题定位残留、全局 accessibility、截图、小屏布局或 VoiceOver 实机验收。

## 实现说明

- `EventLogView` 新增 `displayMessage(for:)`，唐宋场景下不再直接显示 `entry.message`。
- `turnReportSummary` 的 log highlight 同步走 `displayMessage(for:)`，避免顶部摘要仍放大英文或 raw id。
- 新增私有 `TangSongEventLogMessage` 显示桥，覆盖：
  - 命令接受/驳回、将领军令提交、常见 `CommandValidationError.rawValue`。
  - 选中地块、州府、军队、敌军、非敌对军队、亲征势力切换和目标定位。
  - `attacked/counterattacked/strength/automatic retreat/destroyed` 等战斗日志。
  - `moved to/reinforced/retreated/failed to retreat/completed retreat recovery` 等移动、整补和退却日志。
  - AI command result 与招抚候选跳过摘要。
- 非唐宋场景继续显示原始 `entry.message`，legacy 阿登路径不受影响。

## 并发子 Agent 结果

- EventLog/UI 子 Agent 只读指出 `Text(item.entry.message)` 与战报摘要 highlight 直接裸显原始 message，是当前默认主路径最明确的 EventLog 残留点；同时指出 `GameLogEntry.message` 只有最终文案，短期适合 UI 显示桥，中期再考虑结构化 payload。
- MapEditor 子 Agent 只读指出 MapEditor 默认文档名、资源 fallback、状态栏、Picker、导出错误、地块/州府/方面 id、兵种缩写和导出注记仍有英文或 raw id 残留，建议后续拆成独立 v5.8e MapEditor 切片。
- Docs/QA 子 Agent 只读指出 README、plan、flow、update_log 顶部仍有历史二战/旧版本定位残留，建议后续 docs-only 收口默认产品定位和 v5.8 RC 路线。

## 验证口径

- 按用户当前要求，本轮不在本机运行测试、build、Swift parse、Markdown 检查或 `git diff --check`。
- 本轮完成后直接 commit 并 push 到 `origin/main`，触发 GitHub Actions `WWIIHexV0 CI Results`。
- Agent C 验收必须下载最新 `origin/main` commit 对应的未加密 artifact，核对 `ci-artifact-manifest.json` 的 `branch`、`commitSha`、`runId`、`runAttempt`，并阅读 `junit.xml`、`static-checks.log`、`xcodebuild.log` 和 `ci-failure-summary.md`。

## 遗留风险

- `EventLogView` 仍是字符串解析型显示桥，覆盖高频日志，不等于完整结构化战报系统。
- `Command.displayName(isTangSongScenario:)` 仍可能包含单位 id、region id 或 country id；后续可引入 state-aware command display。
- `GameLogEntry` 仍没有 `displayKey/payload`，中期需要结构化日志字段或复用 `CommandResultLogEntry`。
- MapEditor 英文错误、README/flow/update_log 历史定位残留、全局 accessibility、截图、小屏布局和 VoiceOver 验收仍未完成。
